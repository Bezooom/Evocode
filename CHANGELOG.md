# История изменений (Changelog) проекта «Эвокод»

Все значимые изменения в проекте «Эвокод» будут документироваться в этом файле.
Формат ведения истории изменений основан на принципах [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

---

## [0.9.0] — 2026-07-20 (Release Candidate 1)

Этот релиз является важной вехой: мы закрыли все задачи фазы **F3 (Hardening РФ)** и перешли в режим кандидат-релиза. Система переориентирована с классической среды разработки на **специализированное рабочее место оператора** с максимальным фокусом на безопасность данных (DLP) и изоляцию от исходного кода.

### Добавлено
- **Визуальный режим просмотра документов (Operator Mode):**
  - Добавлен встроенный Custom Text Editor Provider для автоматического открытия файлов `*.html` и `*.md` в виде интерактивного браузерного превью (Webview).
  - Для HTML-файлов динамически внедряется директива `<base href="...">` для корректной подгрузки локальных таблиц стилей (CSS), шрифтов и картинок без утечек во внешнюю сеть.
  - Для Markdown-документов автоматически генерируется стилизованная веб-страница, адаптированная под цветовую схему Midnight Fusion.
  - Оператор больше не взаимодействует с кодом разметки напрямую (при необходимости исходный код можно открыть через контекстное меню «Открыть с помощью...»).
- **Умный роутер бесплатных моделей OpenRouter (`openrouter-auto`):**
  - Реализован алгоритм анализа текста входящего запроса на лету.
  - Запросы с кодом автоматически перенаправляются на высокопроизводительную модель `qwen/qwen-2.5-coder-32b-instruct:free`.
  - Текстовые и аналитические запросы направляются на быструю `google/gemini-2.5-flash`.
  - Выбор этой опции в настройках сводит затраты на API к нулю при сохранении максимальной эффективности.
- **Поддержка авторизации и лимитов в Core:**
  - Реализована проверка токена Bearer авторизации для сетевых клиентов.
  - Добавлен Rate Limiter для предотвращения DDoS/перегрузки локального сервера Core.
  - Настройка автоматического связывания Core исключительно с сетевым адресом `127.0.0.1` по умолчанию.
- **Безопасность загрузки навыков (Skill Sync):**
  - Добавлены проверки относительных путей для предотвращения выхода из доверенной директории (Path Traversal).
  - Реализована сверка SHA-256 контрольных сумм импортируемых файлов.
  - Введено ограничение перенаправлений HTTP-запросов (SSRF-защита) при получении внешних манифестов.

### Изменено
- **Локализация шрифтов (100% Offline-режим):**
  - Полностью удалены CDN-ссылки на Google Fonts API.
  - Веб-панели и встроенные редакторы переведены на использование безопасных локальных системных шрифтов (`system-ui`, `-apple-system`, `sans-serif`).
- **Синхронизация дизайн-системы Midnight Fusion:**
  - Актуализирован файл `DESIGN_SYSTEM.md` с переходом на фиолетово-индиговую цветовую гамму.
  - Скорректировано отображение значка «Настройки» на панели Activity Bar.
  - Исправлено наложение векторного логотипа в брендированных иконках (выровнена геометрия и исправлена невидимая градиентная линия).
- **Исправление багов инференса:**
  - Устранена ошибка `evocode-auto is not a valid model ID`. Виртуальные идентификаторы моделей теперь автоматически транслируются сервером Core в целевые физические модели инференса в зависимости от выбранного режима.

### Упаковка и сборка
- Обновлены конфигурационные скрипты для сборки официальных пакетов с версией `0.9.0`.
- Собраны дистрибутивы `evocode_0.9.0_amd64.deb` и `Evocode-0.9.0-x86_64.AppImage`.
- Успешно пройдены сквозные интеграционные тесты запуска среды (smoke-тесты).

## [0.5.0] — 2026-07-19 (Early Beta)

### Добавлено
- Первоначальное брендирование VSCodium под имя «Эвокод».
- Цветовая тема оформления **Midnight Fusion**.
- Локальный Core-сервер с поддержкой локального инференса.
- Скрипты упаковки портативной версии IDE, а также пакетов `.deb` и `.AppImage`.
