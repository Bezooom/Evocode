# История изменений (Changelog) проекта «Эвокод»

Все значимые изменения в проекте «Эвокод» будут документироваться в этом файле.
Формат ведения истории изменений основан на принципах [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

---

## [0.95.0] — 2026-07-20 (Release Candidate 2)

Второй кандидат-релиз: **Skill Router v2 (M1–M4)** и **dual-model FIM** (лёгкая Neurocontrol-модель для autocomplete).

### Добавлено
- **Skill Router v2**
  - Multi-dialect frontmatter, index, ranking (triggers / packs / tier / mutex / persona cap).
  - Seed pack `evocode-*` (privacy, operator, local LLM, DLP, authoring) с RU-triggers.
  - Product panel: dry-run route, pack toggles, lab switch, reindex.
  - Corpus hygiene CLI (`skills:m3:*`): top-level triggers, pack tags, quarantine dups.
  - Hybrid embeddings (M4): hash/sqlite-vec + optional inference backend; `routeAsync` / `reindexAll`.
  - API: `POST /v1/skills/route`, `POST /v1/skills/reindex`, skills section in `/v1/config`.
- **Dual-model FIM / autocomplete**
  - Профиль `fim-small` → GGUF из Neurocontrol (~2G), **CPU `-ngl 0`**, порт **8082**.
  - `POST /v1/completions` и `/v1/fim` → лёгкая модель; model id **`evocode-fim`**.
  - `LLAMA_FIM_ENABLED=true` по умолчанию; health/runtime: `fimReady`.
  - UI: бейджи chat/FIM, toggle FIM, автостарт `autoStartFimModel` после coder.
  - Defaults autocomplete: provider `evocode`, model `evocode-fim`.

### Изменено
- Версия продукта **0.9.0 → 0.95.0** (RC2).
- Документация: RUNTIME dual-model, SKILL_AUTHORING, OPENAPI, STATUS.

### Упаковка
- `VERSION=0.95.0` в `package-deb` / `package-appimage`.
- Артефакты: `evocode_0.95.0_amd64.deb`, `Evocode-0.95.0-x86_64.AppImage` (после `ide:package-*`).

## [0.9.0] — 2026-07-20 (Release Candidate 1)

Этот релиз является важной вехой: мы закрыли все задачи фазы **F3 (Hardening РФ)** и перешли в режим кандидат-релиза. Система переориентирована с классической среды разработки на **специализированное рабочее место оператора** с максимальным фокусом на безопасность данных (DLP) и изоляцию от исходного кода.

### Добавлено
- **Визуальный режим просмотра документов (Operator Mode):**
  - Добавлен встроенный Custom Text Editor Provider для автоматического открытия файлов `*.html` и `*.md` в виде интерактивного браузерного превью (Webview).
  - Для HTML-файлов динамически внедряется директива `<base href="...">` для корректной подгрузки локальных таблиц стилей (CSS), шрифтов и картинок без утечек во внешнюю сеть.
  - Для Markdown-документов автоматически генерируется стилизованная веб-страница, адаптированная под цветовую схему Midnight Fusion.
  - Оператор больше не взаимодействует с кодом разметки напрямую (при необходимости исходный код можно открыть через контекстное меню «Открыть с помощью...»).
- **Claude-style временные артефакты (авто-превью и очистка):**
  - Интегрировано автоматическое выявление тегов `<artifact>` в ответах ассистента. Они сохраняются во временной папке `~/.config/evocode/artifacts/` и открываются на правой панели в режиме превью без фокуса, оставляя курсор оператора в чате.
  - Внедрена автоматическая очистка директории временных артефактов при каждом запуске IDE.
- **Умный роутер бесплатных моделей OpenRouter (`openrouter-auto`):**
  - Подключено автоматическое перенаправление запросов на динамический роутер `openrouter/free`.
  - OpenRouter самостоятельно подбирает наиболее подходящие и доступные в данный момент бесплатные модели (например, Nemotron 3 Ultra для рассуждений или Poolside Laguna для кодинга) с нулевой стоимостью токенов.
  - Это обеспечивает стопроцентную отказоустойчивость при ротации или временном отключении конкретных бесплатных моделей со стороны провайдеров.
- **Универсальный локальный Fallback:**
  - Настроено автоматическое обнаружение любых ошибок на облачном маршруте (статус `>= 400`, включая лимиты кредитов `402`, ошибки моделей `404` и сбои облака `5xx`) с мгновенным прозрачным переключением на локальный LLM инференс без вывода ошибок оператору.
- **Авто-компакт сессий (Session Compaction):**
  - Реализовано уплотнение истории сообщений по алгоритму скользящего окна при превышении лимита в `250 000` токенов, сохраняя системный контекст, навыки и последний запрос.
- **Поддержка авторизации и лимитов в Core:**
  - Реализована проверка токена Bearer авторизации для сетевых клиентов.
  - Добавлен Rate Limiter для предотвращения DDoS/перегрузки локального сервера Core.
  - Настройка автоматического связывания Core исключительно с сетевым адресом `127.0.0.1` по умолчанию.
- **Безопасность загрузки навыков (Skill Sync):**
  - Добавлены проверки относительных путей для предотвращения выхода из доверенной директории (Path Traversal).
  - Реализована сверка SHA-256 контрольных сумм импортируемых файлов.
  - Введено ограничение перенаправлений HTTP-запросов (SSRF-защита) при получении внешних манифестов.

### Изменено
- **Локализация шрифтов (100% Offline-режим):**
  - Полностью удалены CDN-ссылки на Google Fonts API.
  - Веб-панели и встроенные редакторы переведены на использование безопасных локальных системных шрифтов (`system-ui`, `-apple-system`, `sans-serif`).
- **Синхронизация дизайн-системы Midnight Fusion:**
  - Актуализирован файл `DESIGN_SYSTEM.md` с переходом на фиолетово-индиговую цветовую гамму.
  - Скорректировано отображение значка «Настройки» на панели Activity Bar.
  - Исправлено наложение векторного логотипа в брендированных иконках (выровнена геометрия и исправлена невидимая градиентная линия).
- **Скрыты технические маркеры `[PLAN]` и `[EXECUTE]`:**
  - В промпт добавлены правила оформления ответов, запрещающие вывод служебных заголовков в чат.
- **Полная изоляция от Kilo настроек:**
  - Из конфигураций удален путь `~/.config/kilo/skills`. Навыки копируются из рабочей директории в `~/.config/evocode/skills` при каждом запуске IDE.
- **Исправление багов инференса:**
  - Устранена ошибка `evocode-auto is not a valid model ID`. Виртуальные идентификаторы моделей теперь автоматически транслируются сервером Core в целевые физические модели инференса в зависимости от выбранного режима.
