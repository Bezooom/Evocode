# ГОСТ 19.101-2024 — Программное обеспечение

## Предисловие

РАЗРАБОТАН АО «ВНИИС» и ООО «ИАВЦ»
ВНЕСЕН Федеральным агентством по техническому регулированию и метрологии
ПРИНЯТ Межгосударственным советом по стандартизации

## Термины и определения

| Термин | Определение |
|--------|-------------|
| Программный компонент | Программа, рассматриваемая как единое целое, выполняющая законченную функцию |
| Программный комплекс | Программа, состоящая из двух или более программных компонентов и/или программных комплексов, выполняющих взаимосвязанные функции |
| Комплекс программ | Совокупность программ, выполняющих не взаимосвязанные или не непосредственно связанные функции |
| Вид программного документа | Вид документа, регламентирующего состав, содержание, порядок разработки и оформления документов на программные изделия |
| Состав программы и документации | Состав программы и документации на нее, устанавливаемый для данного программного изделия в техническом задании |
| Ведомость держателей подлинников | Перечень предприятий, на которых хранятся подлинники программных документов |
| Символическая запись программы | Символьная запись программы и дополнительные описания и комментарии (при необходимости) |
| Сведения о программном изделии | Сведения о функциональном назначении, логической структуре и функционировании программы |
| Программа и методика испытаний | Требования к программе и программной документации, подлежащие проверке при испытаниях, а также средства, порядок и методы испытаний |
| Техническое задание | Основания для разработки программы, назначение разработки, требования к программе, требования к программной документации, технико-экономические показатели, стадии и этапы разработки, порядок контроля и приемки |
| Пояснительная записка | Назначение и область применения программы, технические характеристики, ожидаемые технико-экономические показатели, источники, использованные при разработке |
| Эксплуатационные документы | Основные характеристики программы, сведения, необходимые для обеспечения функционирования, эксплуатации и выполнения программы |
| Вид эксплуатационного документа | Содержание эксплуатационного документа |
| Ведомость эксплуатационных документов | Перечень эксплуатационных документов на программу |
| Основные характеристики программы | Основные характеристики программы, комплектность и сведения об эксплуатации программы |
| Описание применения | Сведения о назначении программы, области применения, классе решаемых задач, ограничениях для применения, минимальной конфигурации технических средств |
| Руководство системного программиста | Сведения для проверки, обеспечения функционирования и настройки программы на условия конкретного применения |
| Руководство программиста | Сведения для обеспечения эксплуатации программы |
| Руководство пользователя (оператора) | Сведения для обеспечения выполнения программы |

## Таблица составов документов

| Вид документа | Программный компонент | Программный комплекс | Комплекс программ |
|---------------|----------------------|---------------------|-----------------|
| Ведомость держателей подлинников | О | О | О |
| Ведомость эксплуатационных документов | О | О | О |
| Руководство системного программиста | О | О | О |
| Руководство пользователя (оператора) | О | О | О |
| Описание применения | О | О | О |
| Руководство программиста | О | О | О |

Условные обозначения:
- • — документ обязательный
- О — документ обязательный для программных компонентов, программных комплексов и комплексов программ, имеющих самостоятельное применение
- O — необходимость составления документа определяется на этапе разработки и утверждения технического задания
- - — документ не составляется

## Варианты выполнения ТЗ на ПО (ГОСТ 19.101)

### Вариант В.1
ТЗ на разработку программного изделия, содержащее:
- основания для разработки
- назначение разработки
- требования к программе
- требования к программной документации
- технико-экономические показатели
- стадии и этапы разработки
- порядок контроля и приемки

### Вариант В.2
ТЗ на разработку программного изделия с кратким изложением требований.

### Вариант В.3
ТЗ на разработку программного изделия с расширенным изложением требований.

## Состав программных документов (ГОСТ 19.502)

| Код | Наименование | Стадия |
|-----|-------------|--------|
| ПО | Программа разработки | ТП |
| ОП | Описание программы | ТПР |
| ОПП | Описание применения программы | ПМ |
| ОИ | Объём и состав | ПМ |
| ППИ | Программа и методика испытаний | ПМ |
| ПМ | Пояснительная записка | ПМ |
| Э | Пояснительная записка на эксплуатационные испытания | Эксплуатация |
| А | Акт приёмки программных изделий | ПМ |

## Стадии работ (ГОСТ 19.101)

1. Техническая proposal (ТП) — обоснование целесообразности
2. Техническое задание (ТП) — определение требований к ПО
3. Технический проект (ТПР) — проектная документация
4. Приёмочные испытания (ПМ) — проверка соответствия требованиям
5. Ввод в действие (ТЗ-ВН) — введение ПО в эксплуатацию

## Требования к документации (ГОСТ 19.503)

- Оформление программных документов
- Обозначение документов
- Состав документов
- Правила разработки
- Правила оформления

## Нормоконтроль по ГОСТ 19.101

- Форма ТЗ — по ГОСТ 19.201
- Вариант ТЗ — В.1, В.2 или В.3
- Состав разделов — 10 разделов
- Документирование — по ГОСТ 19.502
- Комплектность — полный комплект
- Гарантийные обязательства — указаны сроки
- Финансирование — указаны источники
- Приложение — справочные данные
