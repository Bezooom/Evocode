# Поиск патентных решений и генерация заявки — Описание продукта (PRD)

## 1. Обзор

Репозиторий — Skill по соглашению [AgentSkills](https://agentskills.io): через `SKILL.md` организован процесс, `prompts/` — шаблоны, `tools/` — скрипты-расширения.

**Цель**: поиск патентных решений → проверка новизны → техническая заявка (обезличенный шаблон) → самопроверка и итеративные исправления.

## 2. Процесс пользователя

```
Пользователь (natural language / косвенная команда)
        ↓
Шаг 1  Границы и ввод (prompts/intake.md)
        ↓
Шаг 2  Сканирование проекта (project_scan.md: .docx/.pptx → MD; независимые изображения — пропустить)
        ↓
Шаги 3–4  Кандидаты + слияние (prompts/patent_points_analyzer.md)
        ↓
Шаг 5  Проверка новизны (prior_art_search.md: **приоритетно** ФИПС (Роспатент), при ошибке — WebSearch)
        ↓
Шаг 6  Предпросмотр (prompts/disclosure_preview.md)
        ↓
Шаг 7  Полная заявка (prompts/disclosure_builder.md + template_reference.md)
        ↓
Шаг 8  Самопроверка (prompts/disclosure_self_check.md) → исправление и доставка
        ↓
Доставка  Каталог пользователя .md/.docx: **название + timestamp** (§7.3 пункт 5, первичная и итерации, не перезаписывать по умолчанию)
        ↓
Итерация  Намерение изменить: слияние (merger.md) / исправление (correction_handler.md); то же правило; каталог **`Лог_итераций_заявки.md`** — построчно (время, сводка, файл), см. `iteration_context.md`
```

## 3. Соглашения каталогов

| Путь | Описание |
|------|----------|
| `SKILL.md` | Единственный вход: условия, отображение инструментов, шаги и ссылки на prompts |
| `prompts/` | Пошаговые инструкции, Agent читает и выполняет |
| `tools/` | Опциональные скрипты; `md_to_docx.py`, `docx_to_md.py`, `pptx_to_md.py`, `fips_epub_search.py` (проверка новизны), `fips_epub_crawler.py` / `fips_epub_parse.py`, `iteration_dialog_log.py`; см. `tools/README.md` |
| `docs/` | PRD, архитектурные заметки и т.д. |
| `outputs/` | Каталог доставки; по умолчанию .gitignore |
| `examples/` | Материалы для тренировки (например `example_batch_job_scheduler/knowledge/`); результат — в `outputs/` |

## 4. Ограничения

- Office (Word/PPT): использовать `tools/docx_to_md.py`, `tools/pptx_to_md.py` для конвертации перед сканированием (см. `SKILL.md`).
- Тело заявки **не** содержит чеклиста самопроверки.
- Обезличивание — см. `disclosure_builder.md` / `template_reference.md`.
- Результаты проверки новизны — в раздел I, согласованы с техническими задачами и схемой; источники и библиография — в `prompts/prior_art_search.md`.
- Финальная доставка — **обязательно** .md + .docx; имя **`{название_изобретения}_{YYYYMMDDHHmmss}`** (§7.3 пункт 5, первичная и итерации); `tools/mermaid_render.py` по умолчанию вызывает `md_to_docx.py`; при ошибке Word — разрешить ручной перенос (см. `tools/README.md`).

## 5. Переменные окружения

В Claude Code, OpenClaw и других средах используется **`CLAUDE_SKILL_DIR`** — корень навыка. В Cursor — корень репозитория.
