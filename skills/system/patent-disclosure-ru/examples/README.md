# Каталог примеров

Этот каталог предоставляет **вымышленные материалы для тренировки**, демонстрируя формат ввода **Шага 2 сканирования проекта** (включая演练 Word→Markdown).

| Путь | Описание |
|------|----------|
| `example_batch_job_scheduler/` | Вымышленный сценарий «распределённое планирование пакетных задач и восприятие ресурсов»; **только `knowledge/`** — материалы; патентные решения/проверка/заявка генерировать в `outputs/` |

## Как использовать `example_batch_job_scheduler` для тренировки

Все результаты (патентные решения, чек-лист, предпросмотр, финальная заявка `.md`/`.docx` с timestamp и т.д.) генерируются навыком в **`outputs/{идентификатор_проекта}/`**; правила именования и версий — **`disclosure_builder.md` §7.3**, **`iteration_context.md`**.

### Способ A: только материалы (без Agent)

Откройте `example_batch_job_scheduler/knowledge/`, прочитайте `docs/README.md`, `docs/architecture.md`, `pkg/scheduler/*.go` — поймите, как выглядит **вход для сканирования**.

### Способ B: полная тренировка в Agent (рекомендуется)

Подготовка: навык загружен в Cursor / Claude Code (см. [INSTALL.md](../INSTALL.md) в корне). Вход и шаги — [SKILL.md](../SKILL.md).

1. **Указать путь к проекту**

   `examples/example_batch_job_scheduler/knowledge/`

2. **Запустить навык естественным языком и указать границы** (можно скопировать и отправить Agent):

    ```text
   Выполнить полный процесс patent-disclosure-skill:
   - Каталог сканирования: examples/example_batch_job_scheduler/knowledge/
   - Техническая тема: распределённое планирование пакетных задач, гетерогенные кластеры, восприятие ресурсов и перепланирование с лимитом (см. документы и код в knowledge)
   ```

3. **Пояснение по проверке новизны (важно)**

   Детали — в `prompts/prior_art_search.md`. При тренировке библиографические элементы могут быть плейсхолдерами, структура должна соответствовать этому prompt.

4. **Проверка «результата»**

   - Открыть указанный выходной каталог (например `outputs/пример_каталог/`, весь каталог не коммитится в Git), проверить: патентные решения, чек-лист, предпросмотр, заявка (**Markdown + Word**, имя файла содержит **название + timestamp**).
   - Финальная версия через `tools/mermaid_render.py` (Node.js, `npm install` в `tools/`, `pip install -r requirements.txt`). При ошибке Word — вручную по stderr: `md_to_docx.py`. См. `tools/README.md`.

5. **Версии и итерации**

   Именование доставки и итерации — **`disclosure_builder.md` §7.3 пункт 5**; лог — **`iteration_context.md`**. Опционально `versions/` — не обязательно.

6. **Итеративный режим (по намерению, без фиксированного ключа)**

   Как в [SKILL.md](../SKILL.md): если пользователь явно работает над **готовой заявкой** — **добавление материалов** или **исправление/корректировка** — Agent должен **`Read`** `prompts/iteration_context.md` и `prompts/merger.md` или `prompts/correction_handler.md`.

   Пример:

    ```text
   В существующей заявке outputs/.../название_изобретения_timestamp.md:
   - Объединить новые примеры из приложения (merger); или исправить несоответствие 3.5 формулам текста (correction_handler)
   ```
