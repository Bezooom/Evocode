# Сканирование проектной документации (Шаг 2)

## Цель

Сканировать и извлечь **патентоспособный** контент с приоритизацией. **Адаптируйте пути сканирования под текущую структуру проекта.**

## Таблица приоритетов

| Приоритет | Тип документа | Что искать |
|-----------|---------------|-----------|
| 1 | Патентные документы | Анализ патентных решений, существующие заявки, рекомендации по подаче, обобщение инноваций |
| 2 | Документация по проектированию/решениям | Детальное проектирование, обсуждения решений, блок-схемы, полные процессы, сравнительный анализ |
| 3 | Основной код реализации | Алгоритмы и стратегии, бизнес-логика и оркестрация, обработка и трансформация данных, правила и логики решений, API и интеграции, конечные автоматы и планировщики, оптимизация и кэширование, безопасность и авторизация и т.д. (в зависимости от области проекта) |
| 4 | Документация по архитектуре | Проектирование системы, архитектурные описания, разделение модулей, потоки данных и управления |

## Рекомендации по выполнению

- Для крупных репозиториев — сначала поиск/семантический поиск ключевых файлов, затем детальное чтение.
- Фиксируйте **пути или имена файлов** для обезличенных ссылок в заявке (например, «согласно проекту X»).
- При обнаружении **`.docx` / `.pptx`** — **обязательно** по разделу **«Office-документы»** ниже: сначала преобразование в Markdown, затем чтение, нельзя пропускать или читать только текстовую версию.

## Office-документы (.docx / .pptx): обязательное преобразование перед чтением

**Формат**: скрипт поддерживает OOXML (**`.docx` / `.pptx`**) и старые форматы (**`.doc` / `.ppt`**) через unified_converter.

Agent **не должен** игнорировать Word/PPT из-за «удобнее читать текст» — **обязательно** преобразовать в Markdown и включить в сканирование, нельзя читать только `.md` и исходный код.

1. **Обнаружение**: `Glob` или перечислите `*.docx`, `*.pptx`, `*.doc`, `*.ppt` в каталоге сканирования (включая подкаталоги, например `docs/sample_*.docx`).

2. **Преобразование (unified_converter — универсальный инструмент)**: для каждого файла выполните:

   ```bash
   # Универсальный конвертер (поддерживает все форматы)
   python3 ~/.config/kilo/skills/document-converter/tools/unified_converter.py \
       --input "<путь>/<имя>.docx" \
       --output "<тот же каталог или docs>/<имя>.md"
   
   python3 ~/.config/kilo/skills/document-converter/tools/unified_converter.py \
       --input "<путь>/<имя>.pptx" \
       --output "<тот же каталог или docs>/<имя>.md"
   ```

   **Преимущества unified_converter:**
   - Поддержка старых форматов (.doc, .ppt)
   - Автоматический fallback (mammoth → markitdown для DOCX, python-pptx → markitdown для PPTX)
   - Дополнительные форматы: PDF, Excel, Images (OCR), Audio (транскрипция)
   - Сохранены все функции docx_to_md.py и pptx_to_md.py

   **Обратная совместимость**: старые скрипты всё ещё работают:
   ```bash
   # Старый способ (всё ещё работает)
   python3 ${CLAUDE_SKILL_DIR}/tools/docx_to_md.py -i "<путь>/<имя>.docx" -o "<имя>.md"
   python3 ${CLAUDE_SKILL_DIR}/tools/pptx_to_md.py -i "<путь>/<имя>.pptx" -o "<имя>.md"
   ```

   Требуется `pip install -r requirements.txt` (в корне навыка) или `pip install -r ~/.config/kilo/skills/document-converter/requirements.txt`. Рядом создастся **`{имя_md}_media/`** с встроенными изображениями — **основа для сканирования**: текст `.md` и ссылки на изображения.

3. **Чтение**: **`Read`** созданные `.md` (и при необходимости `_media` для обезличенных ссылок), **на равных правах** с `.md` и кодом — собрать в таблицу материалов для патентных решений.

4. **Ключевые точки**: таблицы, нумерованные списки, **заголовки и текст каждой слайда PPT**, **основной текст Word** (без области редактирования), **заметки** (unified_converter записывает в раздел «Заметки докладчика») — всё это источники патентоспособного контента.

5. **Дополнительные возможности unified_converter**:
   - **PDF**: конвертация научных статей, технических документов
   - **Excel**: конвертация таблиц с данными
   - **Images**: OCR для сканов документов
   - **Audio**: транскрипция интервью, совещаний

## Каталог изображений и независимые файлы (пропустить отдельный OCR)

- **`sample_assets/`** и аналогичные каталоги с **независимыми `.png` / `.jpg` / `.webp`**: **не являются** обязательным объектом для пошагового открытия/OCR/описания в Шаге 2 (при дублировании с изображениями внутри Word/PPT тем более не нужно).
- **Исключение**: пользователь **указал** путь к изображению, или изображение **не** в `_media` преобразованных Office и критично для патентных решений — тогда обработать по запросу.
- После преобразования Word/PPT встроенные изображения уже в **`![](относительный_путь)`** — **основной скан — по Markdown**.

Если путь сканирования включает **`examples/example_batch_job_scheduler/knowledge/`** (или аналогичный каталог проекта), охватить:

| Путь | Действие |
|------|----------|
| `docs/architecture.md` | Прямое чтение |
| `docs/sample_architecture_review.docx` | **Сначала** `docx_to_md.py` → затем **Read** созданный `.md` |
| `docs/sample_scheduler_deck.pptx` | **Сначала** `pptx_to_md.py` → затем **Read** созданный `.md` |
| `docs/sample_assets/*.png` | **Пропустить** отдельное детальное чтение (контент уже покрыт встроенными изображениями + MD) |
| `pkg/scheduler/*.go` | Чтение |
