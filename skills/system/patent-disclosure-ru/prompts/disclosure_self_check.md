# Самопроверка после генерации заявки (Шаг 8, внутренняя)

Результат самопроверки — для **исправления тела заявки**; по умолчанию **не** выводить отдельный отчёт; пользователь запросил — предоставить. **Не** включать чеклист в раздел заявки.

## 8.1 Логика и связность

- [ ] Техническое решение образует полный цикл (обнаружение → обработка → расширение/вывод)?
- [ ] Взаимосвязи между модулями ясны? Верхний/нижний уровни согласованы?
- [ ] Для граничных случаев (низкоуверенные примеры и т.д.) есть чёткий путь обработки?
- [ ] Зависимости (классовые центры, динамические пороги и т.д.) имеют обоснование?
- [ ] Раздел V «ключевые моменты» и раздел III «процесс системы» согласованы? Раздел V — тезисы, детали — в разделе III.

## 8.2 Согласование формул и параметров

- [ ] **Формулы**: единообразны по тексту (формулы весов, корректирующих коэффициентов и т.д.)
- [ ] **Диапазоны порогов**: согласованы с текстом (например 0.5–1.5, 0.8–1.2)
- [ ] **Названия параметров**: единообразны, не смешивать синонимы
- [ ] **Числа примеров реализации**: согласованы с разделом 3.5, не противоречивы

## 8.3 Формат и ссылки

- [ ] **Путь итерации**: если выполнен `merger.md` / `correction_handler.md` — есть ли **`## Сводка слияния (архив)`** или **`## Сводка исправлений (архив)`** (согласно этому prompt)?
- [ ] **Лог исправлений**: в каталоге проекта добавлена **`Лог_итераций_заявки.md`** (время записи, сводка, имя доставки, выдержка), см. **`iteration_context.md`**
- [ ] **Имя файла доставки**: все доставленные `.md`/`.docx` — **`{название_изобретения}_{YYYYMMDDHHmmss}.md`** и同名 `.docx` (§7.3 пункт 5, **включая первичную и итерации**), без перезаписи старых
- [ ] **Блок-схемы**: fenced mermaid, финальные PNG через `mermaid_render.py`, **без** ASCII-схем; `.md` доставлен, `.docx` создан или по инструкции
- [ ] Ссылки на разделы: «см. 3.4.1» — существует ли такой раздел?
- [ ] **1.1 Предшествующий уровень техники**: все публикации/статьи имеют **доступную ссылку, согласованную с библиографией** (не вымышленную); согласовано с `prior_art_search.md` и 1.1 «Обзор поиска»
- [ ] **Чек-лист проверки (ФИПС)**: все записи с непустым **`abstract`** — описание схемы и ограничений **на основе понимания abstract** (не только заголовок, не противоречит абстракту); без `abstract` — подтверждено через другие источники
- [ ] **Чистота конца**: **нет** названия репозитория навыка, `examples/`, `disclosure_draft`, «учебный пример», «вымышленный», «не является юридическим обязательством» и других мета-сносок

## Принцип обработки

При обнаружении проблемы — **сразу исправить тело заявки**, затем доставить.
