# Предпросмотр перед генерацией заявки (Шаг 6)

## Цель

Снизить риск возврата: перед полной генерацией показать структурированную выжимку, запросить **подтверждение / корректировку направления**.

Если пользователь явно просит пропустить — сразу перейти к `disclosure_builder.md`.

## Выжимка должна включать

- Название выбранного патентного решения (рабочий заголовок)
- Решаемая техническая задача: соответствующие недостатки предшествующего уровня техники, 1–3 пункта
- Ключевые инновационные модули или этапы (3–6 пунктов)
- Отличие от **наиболее близкого** предшествующего уровня техники (1 абзац)

## Возможный запрос подтверждения

```
Подтвердите направление? После подтверждения будет сгенерирована полная техническая заявка по шаблону, включающая mermaid-диаграммы (при финальной доставке — PNG + .md + .docx).
```

## Запрещено

В конце выжимки Markdown **не** добавлять название репозитория навыка, пути `examples/`, «учебный пример», «вымышленный», «не является юридическим обязательством» и другие мета-сноски (согласно `disclosure_builder.md` к финальной доставке).
