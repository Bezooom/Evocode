# Итеративный режим: исправление по диалогу

## Врата выполнения

Аналогично `merger.md`: сначала **`Read`** **`prompts/iteration_context.md`**, затем чтение текущей версии и намерения исправления; результат — **новый файл** **`{название_изобретения}_{YYYYMMDDHHmmss}.md`** и同名 `.docx` (§7.3 пункт 5), **не** перезаписывать по умолчанию. **Не** возвращаться к Шагу 3–4 полному анализу патентных решений при подтверждённой итерации (если пользователь не просит заново).

## Когда активировать

По **намерению** пользователя: пользователь указывает на **ошибки, несоответствие фактам или параметрам, проблемы формулировок, корректировку объектов охраны** (например «здесь неправильно», «не согласуется с 3.5», «в объектах охраны нужно подчеркнуть XXX») — **выполнить этот процесс**. **Не требуется** слово «итерация»; **не нужно** спрашивать.

## Шаги

1. **Извлечь точки исправления**: конкретный раздел, проблема, ожидаемое — можно цитировать пользователя.
2. **Классификация**:
   - **Факты и техника**: процесс, параметры, взаимосвязи модулей → изменить III и примеры, раздел 3.5.
   - **Проверка новизны и сравнение**: предшествующий уровень или сравнительный анализ неверен → изменить I, при необходимости — дополнительный поиск.
   - **Объекты охраны и формулировки**: разделы IV, V тезисы → согласовать с III, избежать противоречий.
3. **Применение исправлений**: полная версия после исправления — **в новый файл** **`{название_изобретения}_{YYYYMMDDHHmmss}.md`** через `mermaid_render.py`同名 `.docx` (§7.3 пункт 5); **не** перегружать переписыванием не затронутых разделов; **не** перезаписывать имя файла пользователя по умолчанию.
4. **Самопроверка**: выполнить **8.2, 8.3** из `disclosure_self_check.md`.
5. **Лог**: добавить **`Лог_итераций_заявки.md`** (приоритетно **`tools/iteration_dialog_log.py --kind correct`**).

## Сводка исправлений (архив)

**2–5 предложений на русском**: **где изменено**, **на основании чего**, **влияет ли на объекты охраны или результаты проверки**.  
Если не выведено — считать, что prompt не выполнен.
