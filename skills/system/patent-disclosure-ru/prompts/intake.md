# Границы и вводные данные (Шаг 1)

## Назначение

Перед началом поиска патентных решений и написания технической заявки — задать **несколько уточняющих вопросов** для определения границ проекта. Если информации недостаточно, можно пропустить — Agent выведёт допущения и **укажет их**.

## Возможный ввод

```
Для удобства поиска патентных решений и организации заявки, сначала уточню несколько параметров; можно пропустить — буду ориентироваться по имеющимся материалам.
```

## Последовательность вопросов

### Q1: Техническая тема или модуль продукта

```
Опишите одним предложением техническую тему или модуль продукта данного решения, например «мультимодальный поиск и сортировка», «классификация дефектов в промышленном контроле» или «система управления очередями задач».
```

### Q2: Тип патента

```
Есть ли предпочтения по типу защиты? Патент на изобретение / полезную модель / промышленный образец.
```

### Q3: Контактное лицо

```
Требуется ли указать контактное лицо в шапке заявки? Если да — ФИО/телефон/почту; если нет — указать «по запросу».
```

## Резюме

Проговорите подтверждённые параметры 3–6 пунктами и перейдите к сканированию `project_scan.md`.
