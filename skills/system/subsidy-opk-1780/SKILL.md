# Навык: subsidy-opk-1780 — комплект документов на субсидию Минпромторга ОПК

## Когда использовать

Триггеры (явные фразы пользователя или контекст документов):

- «субсидия Минпромторга», «возмещение затрат на внедрение ПО», «субсидия ОПК»
- «приказ №3519», «Решение №2415», «ПП №1780», «ПП №1781»
- «план-график по Приложению №2», «смета по Приложению №3», «заявка на субсидию»
- «отечественное ПО», «внедрение ПО ВВСТ», «ГОЗ + субсидия»
- В папке пользователя есть: «Объявление о проведении отбора…», «Решение о порядке предоставления субсидии», ТЗ на внедрение отечественного ПО

## Что делает навык

Готовит полный комплект из 12 документов (одиннадцать для конверта + заготовка соглашения), строго соответствующий:

- требованиям **пункта 11 Объявления** (требования к участнику),
- требованиям **пункта 13 Объявления** (состав комплекта документов),
- требованиям **пункта 14 Объявления** (сведения об организации),
- формам **Приложений № 1, 2, 3** к Объявлению,
- обязательным **11 блокам ТЗ** (п.13 «г»),
- критериям **a1 – a7** (Приложение № 2 к Решению № 2415).

## Нормативная база

### Основные документы

| Документ | Содержание |
|----------|------------|
| **ПП РФ № 1780** | Правила предоставления субсидий на возмещение части затрат |
| **ПП РФ № 1781** | Правила предоставления субсидий на возмещение части затрат (дополнительно) |
| **Решение Минпромторга № 2415 от 21.05.2025** | Порядок отбора получателей субсидий, критерии a1–a7 |
| **Приказ Минпромторга № 3519 от 21.07.2025** | Объявление о проведении отбора, формы приложений |

### Ключевые положения

- **Одна заявка — один проект.** Можно подать несколько заявок от одной организации (п.15).
- **Изменения и возврат заявки на доработку — не допускаются** (п.18).
- **Срок подписания соглашения победителем — 30 рабочих дней** со дня подписания протокола итогов (п.38).
- **Субсидия по каждому году ≤ 90 %** затрат этого года.

## Обязательные разделы ТЗ (п.13 «г»)

ТЗ на выполнение проекта внедрения отечественного ПО **должно включать**:

1. **Цели и задачи проекта**
2. **Наименование организации — технического заказчика**
3. **Основания для выполнения работ**
4. **Сроки начала и окончания работ**
5. **Порядок оформления, ведения и предъявления результатов работ**
6. **Наименование внедряемого ПО** с указанием **порядкового номера реестровой записи** в едином реестре российских программ для ЭВМ и БД
7. **Функциональные требования к ПО**, в том числе в составе ПАК
8. **Состав, этапность и содержание работ**
9. **Порядок контроля и приёмки результатов работ**
10. **Требования к документированию**
11. **Значения критериев a1 – a7** согласно приложению № 2 к Решению № 2415

> Если в ТЗ нет одного из блоков — **необходимо дополнить** перед подачей (иначе отклонение по п.24 «в» объявления).

## Структура затрат для Плановой сметы (Приложение № 3)

```
ИТОГО_смета = Договор_с_исполнителем  +  ФОТ_своих_спец-тов
            + Страховые_взносы        +  Материалы
            + Амортизация_ОС_и_НМА    +  Командировочные
            + Прочие / накладные

Субсидия = не более 90 % от ИТОГО_смета, по каждому году отдельно
```

### Типовой состав статей сметы

| № | Статья затрат | Источник |
|---|---------------|----------|
| 1 | **ФОТ** штатных специалистов ПАО | Табель + оклады × доля загрузки |
| 2 | **Страховые взносы** с ФОТ (~30,2 %) | ФОТ × 30,2 % |
| 3 | **Материалы и комплектующие** | Бюджет АХО на проект |
| 4 | **Работы и услуги сторонних организаций** | Договор с интегратором |
| 5 | **Приобретение лицензий на ПО** | Счета вендоров |
| 6 | **ПАК** — серверы, СХД, сетевое оборудование | Счета поставщиков |
| 7 | **Амортизация ОС и НМА** | Бухгалтерский расчёт |
| 8 | **Командировочные расходы** | Планы командировок |
| 9 | **Прочие прямые расходы** | Бюджет проекта |
| 10 | **Накладные расходы** | % от прямых |

### Правила заполнения

1. **ФОТ** — на основе табеля специалистов и их окладов, учитывается **фактическая доля занятости** на проекте.
2. **Страховые взносы** — **ФОТ × 30,2 %** (стандартный тариф 2024-2026 гг.).
3. **Работы и услуги сторонних** — **цена договора** с интегратором, распределяется по годам.
4. **Субсидия по каждому году** ≤ 90 % от суммы затрат года.
5. **ИТОГО по годам** в смете = ИТОГО в Таблице 3 Заявки.

## Сквозные сверки значений (критические)

Перед подачей **обязательно** проверить:

- [ ] a1–a7 в Заявке (Табл. 2) = a1–a7 в ТЗ (Табл. 9)
- [ ] **Итого по План-графику** (= стоимость договора с исполнителем) = **строке «Работы и услуги сторонних организаций» в Плановой смете**
- [ ] **Итого по Плановой смете** (ВСЕ статьи) = **Итого в Таблице 3 Заявки**
- [ ] ФОТ штатных специалистов проекта внесён в Смету
- [ ] Страховые взносы с ФОТ внесены (~30,2 %)
- [ ] Прочие статьи (материалы, командировочные, амортизация, накладные) — внесены или явно отсутствуют
- [ ] Субсидия по каждому году ≤ 90 % затрат этого года
- [ ] Даты этапов плана-графика лежат внутри сроков проекта из ТЗ
- [ ] Контракт(ы) предварительных работ упомянуты и в Справке 09, и в «Источниках разработки» ТЗ

## Пошаговый workflow

1. **Изучить требования.** Прочитать `references/announcement-requirements.md` (п.11, п.13, п.14) и `references/tz-mandatory-sections.md` (11 блоков ТЗ).
2. **Собрать контекст.** Заполнить `context.yaml` по образцу: реквизиты заказчика и технического заказчика, данные проекта (наименование, шифр, сроки, ПО, реестровые записи), значения критериев a1–a7, **все статьи сметы**, договор(ы) собственных затрат, контакты.
3. **Верифицировать ТЗ.** Проверить наличие всех 11 обязательных блоков. Блоки, которых нет — добавить перед подачей.
4. **Синхронизировать значения.** Убедиться во всех сквозных сверках (см. выше).
5. **Сгенерировать документы.** Запустить `scripts/build_dossier.py --context <ctx>.yaml --output <out_dir>`.
6. **Сверить по чек-листу.** Прошить, пронумеровать листы, заверить подписями.
7. **Запечатать конверт** и передать в Минпромторг в соответствии с п.21 Объявления.

## Правила и границы

- Навык **не выдумывает** реквизиты (ОГРН/КПП/р/с, номера контрактов ГОЗ). Неизвестные поля — остаются как `___________`.
- Навык **не генерирует**: устав, выписку ЕГРЮЛ, справку ФНС, письмо технического заказчика. Для них формируются титульные листы-слоты с инструкцией.
- Если передан существующий `.docx` ТЗ — скрипт копирует его и добавляет лист проверки по п.13 «г».
- Если ТЗ формируется «с нуля» — рендерится скелет с 11 блоками, требующий заполнения.
- Стиль документов: A4, Times New Roman 14 pt, интервал 1.5, поля 3/1.5/2/2 см, таблицы с сеткой 0.5 pt.
