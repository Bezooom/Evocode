# Академический стиль и лексика для ВКР

## Системные промпты (встроить в AGENTS.md)

### Промпт для генерации научного стиля

```
Ты — строгий научный редактор ВАК. Твоя задача — написать текст для академической работы.

ЗАПРЕЩЕНО:
- Использовать вводные слова-клише («важно отметить», «следует подчеркнуть», «бесспорно»)
- Метафоры, образные выражения, разговорная лексика
- Перечисления ради объёма
- Эмоциональные оценки («удивительно», «поразительно»)
- Слова: «важно», «современный», «безусловно», «очевидно», « несомненно»
- Канцеляризмы: «вышеупомянутый», «рассматриваемый выше»
- ИИ-паттерны: «в современных реалиях», «является ключевым аспектом»

ТРЕБУЕТСЯ:
- Пассивный залог и безличные конструкции
- Строгая терминология по направлению подготовки
- Каждое утверждение — с фактологической или аналитической ценностью
- Ссылки на источники после каждого утверждения
- Логические связки между предложениями
- Тезис → Аргумент → Пример → Вывод

СТИЛЬ:
- Предложения средней длины (20-40 слов)
- Чередование простых и сложных предложений
- Избегать повторяющихся конструкций
- Использовать научные глаголы: «показано», «выявлено», «подтверждено», «установлено»
```

## Академические глаголы

### Вместо разговорной лексики:

| Разговорный | Академический |
|-------------|---------------|
| посмотреть | проанализировать, исследовать, рассмотреть |
| сделать | реализовать, осуществить, провести |
| хороший | эффективный, оптимальный, качественный |
| много | значительное количество, ряд, комплекс |
| показать | продемонстрировать, выявить, иллюстрировать |
| объяснить | интерпретировать, аргументировать, обосновать |
| взять | использовать, применить, задействовать |
| сделать вывод | констатировать, заключить, подчеркнуть |

### Научные глаголы по функциям:

**Анализ:** исследовать, анализировать, изучать, рассматривать, оценивать
**Сравнение:** сопоставлять, соотносить, сравнивать, contrasting, противопоставлять
**Обобщение:** обобщать, систематизировать, классифицировать, синтезировать
**Утверждение:** показывать, демонстрировать, подтверждать, доказывать, устанавливать
**Опровержение:** опровергать, ставить под сомнение, оспаривать, ограничивать

## Логические связки

### Для введения аргумента:
- Как показывает анализ ...
- Согласно исследованию ...
- В работе [X] отмечается, что ...
- Эмпирические данные свидетельствуют о ...

### Для противопоставления:
- Вместе с тем ...
- Однако ...
- Тем не менее ...
- В отличие от ...
- Наряду с этим ...

### Для вывода:
- Таким образом ...
- Следовательно ...
- Из вышеизложенного следует ...
- Полученные результаты позволяют заключить, что ...
- На основании проведённого анализа можно констатировать ...

### Для перехода:
- Изложенный материал позволяет перейти к рассмотрению ...
- Выше были рассмотрены ... что создаёт предпосылки для анализа ...
- Учёт вышеизложенного определяет необходимость исследования ...

**ВАЖНО:** Не использовать «перейдём к» и «далее рассмотрим» в конце параграфов (см. промпт «Сшиватель логики» ниже).

## Структура абзаца

**Правило TEA-V (Тезис-Эvidence-Анализ-Вывод):**

1. **Тезис** (Topic Sentence) — основное утверждение абзаца
2. **Evidence** (Подтверждение) — данные, цитаты, ссылки
3. **Анализ** (Analysis) — интерпретация, связь с тезисом
4. **Вывод** (Conclusion) — итог абзаца, связь со следующей частью

**Пример:**

> Цифровая трансформация предприятий сопровождается изменением организационных структур [1, с. 34]. По данным исследования McKinsey (2023), 73% компаний, внедривших цифровые технологии, отметили снижение бюрократических барьеров [2]. Это подтверждается анализом деятельности ООО «Компания» за 2020–2023 гг.: количество уровней управления сократилось с 6 до 4 (табл. 2.1). Полученные данные свидетельствуют о том, что цифровизация способствует оптимизации организационной структуры.

## Оформление таблиц и рисунков

### Таблицы

**Правила:**
- Заголовок НАД таблицей: «Таблица 2.1 — Динамика показателей»
- Графы таблицы должны иметь заголовки со ссылками на источники
- Примечания ПОД таблицей: «Примечание: составлено автором» или «Примечание: [источник]»
- Ссылка в тексте: «... (табл. 2.1)»
- Таблица не должна обрываться на странице

### Рисунки

**Правила:**
- Подпись ПОД рисунком: «Рис. 2.1 — Структура процесса»
- Источник под подписью: «Источник: составлено автором» или «Источник: [3, с. 45]»
- Ссылка в тексте: «... (рис. 2.1)»
- Использовать векторную графику (SVG, EMF) для чёткости

## Анти-паттерны ИИ-генерации

### Распознаваемые паттерны LLM:

**Фразы-маркеры (исключить):**
1. «В современных реалиях»
2. «Является ключевым аспектом»
3. «Необходимо отметить»
4. «Следует подчеркнуть»
5. «Безусловно, можно утверждать»
6. «Пожалуй, наиболее важным»
7. «Важно понимать, что»
8. «В условиях стремительно развивающегося»
9. «Представляется целесообразным»
10. «Актуальность данного исследования обусловлена»

**Синтаксические паттерны:**
- Чрезмерное использование «является» + существительное
- Повторение одних и тех же вводных слов в начале абзацев
- Идеально сбалансированные предложения одинаковой длины
- Отсутствие специфических деталей и числовых данных

**Стратегия борьбы:**
1. Чередовать длину предложений (10-60 слов)
2. Использовать активный залог в 30-40% предложений
3. Включать конкретные числовые данные
4. Применять разнообразную лексику
5. Добавлять авторские формулировки и оценки

## Статистика стиля для проверки

**Maturity Metrics (измеряются после генерации):**

| Метрика | Бакалавриат | Магистратура |
|---------|-------------|--------------|
| Средняя длина предложения | 20-35 слов | 25-45 слов |
| Разброс длин (std) | 8-15 слов | 10-20 слов |
| Пассивный залог | 20-40% | 30-50% |
| Количество ссылок на абзац | 1-3 | 2-5 |
| Повторяемость n-грамм | < 0.05 | < 0.03 |
| Burstiness (CV длины) | 0.4-0.7 | 0.5-0.8 |

**Burstiness** — коэффициент вариации длин предложений. Высокая burstiness = естественный стиль.
