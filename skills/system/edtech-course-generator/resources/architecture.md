# Архитектура интеграций (Advanced Local-First Media)

Конвейер использует экстремальные оптимизации инференса, перенесенные из High-Performance ML систем (Tail-explorer, Doka), чтобы максимизировать утилизацию RTX 3090 (24GB).

## 1. SGLang & RadixAttention (Кэширование огромного контекста)
- **Движок:** Вместо `vLLM` используется `SGLang` (`python -m sglang.launch_server`).
- **Почему это важно:** Глобальное саммари GraphRAG может достигать 30-50 тысяч токенов. Если запускать параллельную генерацию `CourseManifest`, `PodcastScript` и `PresentationSlides` через обычный API, мы сожжем 150k токенов и получим OOM.
- **TurboCache (RadixAttention):** SGLang нативно кэширует префикс промпта (общий контекст курса). Все параллельные агенты будут использовать **один и тот же закэшированный префикс в VRAM**, вычисляя только дельту (сам промпт агента). Это дает ускорение в 3-5 раз и экономит до 70% видеопамяти.
- **Батчинг:** SGLang позволяет батчить запросы (Continuous Batching), поэтому параллельные запросы от `SlideDeckAgent` и `PodcastAgent` выполняются одновременно и максимально эффективно.
- **Модель:** Qwen 3.5 9B / 14B AWQ или Llama-3-8B.

## 2. Dynamic VRAM Scaling (Hardware-Aware)
Система должна быть адаптивной к ресурсам:
- SGLang запускается с `--mem-fraction-static 0.4` (около 10 GB).
- Оставшиеся 14 GB отдаются `ComfyUI` и `Kokoro`.

## 3. ComfyUI (Генерация Изображений)
- **Назначение:** Иллюстрации для интерактивных презентаций (Визуальных Новелл).
- **Пайплайн:**
  - Клиент отправляет POST-запрос на `http://localhost:8188/prompt` с payload'ом (workflow_api.json).
  - В ноду текстового промпта (обычно ID "6") инъецируется английский промпт от Slide Deck Agent'a.
  - Поллинг через `/history/{prompt_id}` до готовности картинки.
- **Рекомендуемые модели:** Flux.1-schnell FP8 (~12GB VRAM) или SDXL.

## 4. Kokoro-82M TTS (Генерация Подкастов)
- **Назначение:** Озвучка `PodcastScript` двумя голосами (Audio Overview).
- **Пайплайн:**
  - REST API на порту `8880` (`/v1/audio/speech`).
  - Голоса: `am_michael` (мужчина), `af_bella` (женщина).
  - Генерация в байты (`io.BytesIO()`) без сохранения промежуточных файлов на диск для ускорения I/O.
  - Склейка через `pydub.AudioSegment` с паузами 300мс.

## 5. GraphRAG (Hierarchical Retrieval)
- **Community Summaries:** Текст не просто нарезается на куски. Сущности кластеризуются алгоритмом Louvain. Для каждого кластера (Community) пишется промежуточное саммари. 
- Только после этого формируется `global_summary`, что гарантирует сохранение сложных связей и пререквизитов.

## 6. Сборка Презентаций (python-pptx)
- Использование `python-pptx` для маппинга данных в `interactive_quest.pptx`.
- Разметка кнопок (Choices) как MSO_SHAPE.ROUNDED_RECTANGLE с `click_action.target_slide` для создания внутреннего ветвления сюжета (Визуальная новелла).
