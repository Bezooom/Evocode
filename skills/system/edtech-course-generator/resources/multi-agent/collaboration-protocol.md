# Multi-Agent Collaboration Protocol

Внутри навыка `edtech-course-generator` функционирует команда специализированных агентов. Они общаются через Главного Оркестратора.

## 1. Роли Внутренних Агентов
- **Pedagogue Agent (Педагог):** Отвечает за выравнивание по ФГОС/Блуму и построение образовательного пути (Learning Path).
- **Gamification Architect (Игропрактик):** Проектирует механики (Boss Battles, Skill Trees, Narrative Arc) так, чтобы они не были "прикручены сбоку", а органично вплетались в концепцию.
- **Psychologist Agent (Психолог):** Следит за эмоциональной регуляцией. Предотвращает Overjustification Effect и Shame-based motivation. Гарантирует Growth Mindset. На Level 8 симулирует распознавание биофидбека (по времени ответа, количеству исправлений, выбору сложности) и генерирует микро-интервенции (Micro-Interventions) при признаках тревоги или выгорания.
- **Assessment Agent (Оценщик):** Формирует вопросы и `formative_assessment_points`.
- **Narrative Weaver (Сценарист):** Пишет диалоги для `PodcastScript` и истории для `PresentationSlides`, обеспечивая бесшовный сторителлинг (Cross-Modal Coherence).

## 2. Протокол Общения (Debate Protocol)
При генерации сложного курса (или при `Score < 95` от Критика), Оркестратор запускает раунд дебатов:
- Игропрактик может предложить: "Давайте сделаем Босса слишком сложным, чтобы дети проиграли первый раз".
- Психолог возразит: "Для когорты 7-9 лет это вызовет сильную фрустрацию. Предлагаю добавить 'волшебного помощника'."
- Педагог зафиксирует, что этот конфликт помогает закрыть цель: "Умение принимать помощь".

## 3. Синергия
Результатом совместной работы является полностью интегрированный `CourseManifest` и набор артефактов, где каждое решение обосновано с нескольких точек зрения.
