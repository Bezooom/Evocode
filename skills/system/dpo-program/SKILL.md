# ДПО Программа — Разработка программ дополнительного профессионального образования

Агентская система для разработки рабочих программ ДПО в соответствии с ФЗ-273,
Приказом Минпросвещения РФ №731, ГОСТ Р 7.0.100-2018.
Покрывает полный цикл от концепции до утверждения программы.

---

## Нормативная база

### Ключевые документы

1. **ФЗ-273** «Об образовании в РФ» от 29.12.2012, ст. 76 (ДПО)
2. **Приказ Минпросвещения №731** от 17.05.2023 «Порядок организации и осуществления
   образовательной деятельности» (заменил Приказ 374 Минобразования)
3. **Приказ Минпросвещения №352** от 17.05.2023 «Рабочая программа обучения»
4. **ГОСТ Р 7.0.100-2018** — библиографические записи
5. **ГОСТ Р ИСО 9001-2015** — система менеджмента качества (для аккредитованных организаций)
6. **Письмо РАНО** о структуре рабочей программы ДПО

### Критерии программ ДПО

| Тип программы | Мин. объём | Документ |
|---|---|---|
| Повышение квалификации | 16 академ. часов | Удостоверение о повышении квалификации |
| Профессиональная переподготовка | 250 академ. часов | Диплом о профессиональной переподготовке |
| Семинар/тренинг | от 1 часа | Сертификат участника |
| Стажировка | от 16 часов | Справка о прохождении стажировки |

### Формы обучения ДПО

- Очная (с отрывом от работы)
- Заочная (с применением ЭО/ДОТ)
- Очно-заочная (вечерняя, выходные)
- Смешанная

## Фаза 1: Инициализация проекта

### 1.1 Сбор требований

Собрать у пользователя:

1. **Тип программы**: повышение квалификации (ПК) или переподготовка (ПП)
2. **Направление/код специальности**: ОКПД2 / ОКЗ / ОКСП
3. **Название программы** (черновое)
4. **Целевая аудитория**: гос. служащие, бюджетники, коммерческий сектор, ИП
5. **Объём в академических часах**: 16-72 (ПК), 250-500 (ПП)
6. **Форма обучения**: очная / заочная / смешанная
7. **Срок реализации**: 1 месяц, 3 месяца, 6 месяцев, 12 месяцев
8. **Наличие утверждённого плана-графика**: да/нет
9. **Требования лицензирующего органа**: особые требования (если есть)
10. **Финансирование**: бюджет, контракт, смешанное

## Фаза 2: Концепция программы

### 2.1 Обоснование актуальности (`concept/rationale.md`)

**Структура:**

1. **Контекст** — состояние отрасли/сферы на текущий момент
2. **Проблема** — какие компетенции отсутствуют/устарели
3. **Решение** — как программа закрывает пробел
4. **Целевые показатели** — статистика (рынок труда, запросы работодателей)
5. **Соответствие стратегическим документам** — отраслевые стратегии, нацпроекты

**Пример формулировки:**

> «В условиях цифровой трансформации экономики 67% специалистов бюджетной сферы
> не обладают компетенциями в области электронного документооборота, что снижает
> эффективность муниципальных услуг на 23% (данные Росстата, 2025).»

### 2.2 Целевая аудитория (`concept/target-audience.md`)

**Описание ЦА:**

- Должность/сфера деятельности
- Требуемый уровень образования
- Предварительные знания и навыки
- Средний возраст, стаж работы
- География (регион/федеральный округ)

**Допуск к обучению** (согласно ФЗ-273, ст. 76):
- Для ПК: любое уровень образования (СПО, ВО)
- Для ПП: среднее профессиональное или высшее образование

### 2.3 Положение о программе (`concept/charter.md`)

**Разделы положения:**

1. Общие положения (назначение, нормативная база)
2. Цели и задачи программы
3. Планируемые результаты (по компетенциям)
4. Объём программы в часах
5. Формы обучения и аттестации
6. Документ, выдаваемый по окончании
7. Финансовые условия
8. Ответственность сторон

## Фаза 3: Рабочая программа

### 3.1 Структура рабочей программы ДПО

Согласно Приказу Минпросвещения №352:

```
1. Паспорт программы
   1.1. Наименование программы
   1.2. Код специальности/направления
   1.3. Целевая аудитория
   1.4. Объём программы (академ. часы)
   1.5. Форма обучения
   1.6. Документ о выдаче

2. Планируемые результаты
   2.1. Формируемые компетенции (硬/soft skills)
   2.2. Результаты обучения (знает, умеет, владеет)
   2.3. Критерии оценки

3. Содержание программы
   3.1. Раздел/модуль 1 — наименование, объём, содержание
   3.2. Раздел/модуль 2 — ...
   ...

4. Фонды оценочных средств

5. Учебно-методическое обеспечение
   5.1. Основная литература
   5.2. Дополнительная литература
   5.3. Интернет-ресурсы

6. Кадровое обеспечение
   6.1. Преподаватели (квалификация)
   6.2. Эксперты-практики

7. Информационно-техническое обеспечение

8. Финансово-экономические условия

9. Календарный план
```

### 3.2 Паспорт программы (`program/working-program.md`)

**Паспорт заполняется в таблице:**

| Параметр | Значение |
|---|---|
| Полное наименование | «Название программы» |
| Тип программы | Повышение квалификации / Переподготовка |
| Код по ОКСП | XXXX.XX.XX |
| Целевая аудитория | Категория слушателей |
| Объём (акад. часы) | XX |
| Форма обучения | Очная / Заочная / Смешанная |
| Сроки реализации | Х месяцев |
| Стоимость | XXXXXX руб. |
| Выдаваемый документ | Удостоверение / Диплом |

### 3.3 Планируемые результаты (`program/content.md`)

**Формулировка компетенций (Bloom's taxonomy):**

| Компетенция | Индикатор | Уровень |
|---|---|---|
| ПК-1 (профессиональная) | Применяет методы ... | уровень 3 (применение) |
| ПК-2 | Анализирует данные ... | уровень 2 (понимание) |
| ИК-1 (общие) | Работает в команде | уровень 1 (знание) |

**Результаты по формуле «знает-умеет-владеет»:**

- **Знает**: основные принципы, нормативную базу, классификации
- **Умеет**: применять методы, анализировать данные, принимать решения
- **Владеет**: навыками, технологиями, инструментами

### 3.4 Содержательная часть — модули

**Для каждой программы модули/разделы:**

```markdown
## Модуль N. Название модуля (X акад. часов)

### Цели модуля
[Что слушатель достигнет]

### Содержание
- Тема 1.X: ... (X часов)
- Тема 1.Y: ... (X часов)
- Практическое занятие 1.Z: ... (X часов)

### Формы занятий
- Лекция (X ч.) — ...
- Семинар (X ч.) — ...
- Практическое занятие (X ч.) — ...
- Самостоятельная работа (X ч.) — ...
```

**Баланс форм (рекомендации):**

| Форма | ПК (16-72 ч.) | ПП (250+ ч.) |
|---|---|---|
| Лекции | 10-20% | 15-25% |
| Семинары | 15-25% | 10-15% |
| Практика | 30-40% | 25-35% |
| Самост. работа | 20-30% | 30-40% |

### 3.5 Учебный план (`program/curriculum.md`)

**Таблица учебного плана:**

| № | Раздел/модуль | Всего часов | В т.ч. | Форма контр. |
|---|---|---|---|---|
| | | | Ауд. | Самост. | |
| 1 | Модуль 1 | X | X | X | зачёт |
| 2 | Модуль 2 | X | X | X | тест |
| ... | | | | | |
| N | Итоговая аттестация | X | — | X | экзамен |

**Аудиторные часы** — занятия с преподавателем (лекции, семинары, практика)
**Самостоятельная работа** — домашние задания, проекты, чтение

## Фаза 4: Учебно-методический комплект (УМК)

### 4.1 Структура УМК (`umk/umk-full.md`)

```
УМК программы ДПО:
├── Для слушателя:
│   ├── Учебно-методическое описание
│   ├── Программа дисциплины
│   ├── Рабочая программа учебного курса
│   ├── Методические указания по выполнению работ
│   ├── Сборник заданий / кейсов
│   └── Рекомендации по самостоятельной работе
├── Для преподавателя:
│   ├── Рабочая программа
│   ├── Календарно-тематический план
│   ├── Методические указания
│   ├── Фонды оценочных средств
│   └── Примерный перечень вопросов к зачёту/экзамену
└── Оценочные средства:
    ├── Фонды ОС (тесты, кейсы, задания)
    ├── Критерии оценивания
    └── Протоколы аттестации
```

### 4.2 Методические указания для преподавателя (`umk/teacher-guide.md`)

**Содержание:**

1. Цели и задачи модуля
2. Планируемые результаты (в терминах компетенций)
3. Типовые темы занятий
4. Рекомендуемые формы проведения
5. Методические рекомендации по проведению
6. Рекомендации по самостоятельной работе
7. Контроль и оценка результатов
8. Рекомендуемые ресурсы

### 4.3 Фонды оценочных средств (`umk/assessment/`)

**Для каждой темы/модуля:**

- Тестовые задания (минимум 20 вариантов для ПК, 50 для ПП)
- Кейс-задачи (практические ситуации)
- Практические задания
- Промежуточная и итоговая аттестация

**Критерии оценивания:**

| Оценка | Критерий |
|---|---|
| Отлично (5) | 85-100% правильных ответов |
| Хорошо (4) | 70-84% |
| Удовлетворительно (3) | 50-69% |
| Неудовлетворительно (2) | менее 50% |

## Фаза 5: Календарный план и организация

### 5.1 Календарный план (`program/timeline.md`)

**Разбить программу на этапы:**

| Неделя | Тема | Форма | Часы | Результат |
|---|---|---|---|---|
| 1 | Модуль 1.1 | Лекция + семинар | 8 | Знание основ |
| 2 | Модуль 1.2 | Практикум | 8 | Навык применения |
| ... | | | | |
| N | Итоговая аттестация | Экзамен | 4 | Оценка компетенций |

### 5.2 Организация учебного процесса

**Для очной формы:**
- Аудиторный фонд (мультимедиа, интерактивные доски)
- Расписание занятий
- Правила посещения (минимум 80% для ПК, 90% для ПП)

**Для заочной формы:**
- Платформа ЭО (Moodle, LMS, собственная)
- Структура курса на платформе
- Формы взаимодействия с преподавателем
- Сроки сдачи заданий

## Фаза 6: Договоры и документы

### 6.1 Шаблон договора (`docs/contract-template.md`)

**Разделы договора ДПО:**

1. Предмет договора (наименование программы, объём в часах)
2. Права и обязанности сторон
3. Стоимость и порядок оплаты
4. Формы обучения и контроля
5. Документ, выдаваемый по окончании
6. Ответственность сторон
7. Форс-мажор
8. Споры и порядок их разрешения
9. Реквизиты и подписи сторон

### 6.2 Форма заявки (`docs/application-form.md`)

**Для физических лиц:**
- ФИО, дата рождения, паспортные данные
- Образование (документ, год, номер записи в ЗАГС)
- Место работы, должность
- Контактные данные
- Программы обучения (выбор из перечня)

**Для юридических лиц:**
- Данные организации (ИНН, ОГРН, адрес)
- Список слушателей (ФИО, должность, образование)
- Форма оплаты, реквизиты
- Контактное лицо

### 6.3 Шаблоны документов об обучении (`docs/certificate-template.md`)

**Удостоверение о повышении квалификации:**
- Наименование организации-вуза
- ФИО слушателя
- Наименование программы
- Объём в академических часах
- Дата выдачи
- Номер записи в журнале
- Подпись руководителя, печать

**Диплом о профессиональной переподготовке:**
- Наименование организации-вуза
- ФИО выпускника
- Наименование программы
- Объём в академических часах (не менее 250)
- Приобретаемая компетенция/квалификация
- Дата выдачи, номер
- Подпись, печать, голограмма

## Фаза 7: Нормоконтроль и утверждение

### 7.1 Чек-лист нормоконтроля

| Параметр | Требование | Проверка |
|---|---|---|
| Код по ОКСП | Соответствует направлению программы | [ ] |
| Объём (ПК) | Не менее 16 акад. часов | [ ] |
| Объём (ПП) | Не менее 250 акад. часов | [ ] |
| Формулировки результатов | По Bloom's taxonomy | [ ] |
| Баланс часов | Ауд. + самост. = общий объём | [ ] |
| Фонды ОС | Соответствуют каждому модулю | [ ] |
| Литература | Не старше 5 лет (не менее 30% за 3 года) | [ ] |
| Квалификация преподавателей | Наличие учёной степени / опыта | [ ] |
| Оформление | ГОСТ Р 7.0.100-2018 | [ ] |
| Номер лицензии | Указан в программе | [ ] |

### 7.2 Экспертная оценка программы

**Комиссия включает:**

1. Эксперт по содержанию (предметная область)
2. Методист (педагогический дизайн)
3. Представитель работодателя (отраслевой запрос)
4. Юрист (соответствие ФЗ-273)

**Критерии экспертизы:**

| Критерий | Вес | Описание |
|---|---|---|
| Актуальность | 0.25 | Соответствие запросам рынка |
| Полнота | 0.20 | Охват всех компетенций |
| Методическая грамотность | 0.20 | Соответствие педагогическим стандартам |
| Практическая значимость | 0.20 | Применимость в работе |
| Оформление | 0.15 | Соответствие ГОСТ |

**Минимальный порог:** 4.0 из 5.0 для допуска к утверждению

## Фаза 8: Публикация и мониторинг

### 8.1 Публикация в перечне программ

**Требования к публикации:**

1. Наименование программы (полное и сокращённое)
2. Код специальности / направление
3. Объём в часах
4. Стоимость
5. Форма обучения
6. Срок реализации
7. ФИО руководителя программы
8. Контактная информация

### 8.2 Мониторинг качества

**Обратная связь от слушателей:**
- Анкета удовлетворённости (после каждого модуля)
- Итоговая оценка программы
- Готовность рекомендовать коллегам (NPS)

**Аналитика:**
- Процент завершивших программу
- Средний балл аттестации
- Динамика трудоустройства / повышения выпускников
- Количество отзывов и благодарностей

## Системные промпты

### Промпт для генерации содержательной части модуля

```
Ты — методист ДПО. Разработай содержательную часть модуля программы
дополнительного профессионального образования.

Дано:
- Название модуля: [название]
- Объём: [X] академических часов
- Целевая аудитория: [аудитория]
- Формируемые компетенции: [компетенции]

Требования:
1. Разбить на темы с указанием часов на каждую
2. Для каждой темы определить форму (лекция/семинар/практика)
3. Сформулировать результаты обучения по Bloom's taxonomy
4. Указать виды самостоятельной работы
5. Проверить баланс: аудиторные часы + самост. = общий объём
6. Привязать каждую тему к формируемым компетенциям

Выход: Markdown-таблица с темами + текстовое описание содержания
```

### Промпт для генерации тестовых заданий

```
Ты — разработчик контрольных материалов ДПО.

Создай 10 тестовых вопросов с 4 вариантами ответов по теме: [тема]

Требования:
- Вопросы разного уровня (знание, понимание, применение)
- Каждый вариант — правдоподобный, один правильный
- К правильному ответу — ссылка на источник
- Формат: JSON с полями question, options[], correct_index, source

Используй таксономию Блума: 30% знание, 40% понимание, 30% применение
```

### Промпт для нормоконтроля

```
Ты — эксперт по нормоконтролю программ ДПО.

Проверь рабочую программу на соответствие:
1. ФЗ-273 ст. 76 (минимальный объём, допуск)
2. Приказ Минпросвещения №352 (структура программы)
3. Баланс часов (аудиторные + самостоятельные = заявленные)
4. Таксономия Блума (формулировки результатов)
5. Актуальность литературы (не старше 5 лет)

Результат: JSON с полями:
- passed: boolean
- issues: [{line, type, severity, description}]
- warnings: [{line, type, description}]
- recommendations: [текст]
```

## Антипаттерны ДПО-программ

1. **Объём менее 16 часов для ПК** — программа не соответствует ФЗ-273
2. **Объём менее 250 часов для ПП** — диплом не выдаётся
3. **Результаты без привязки к компетенциям** — «слушатель узнает о...» вместо «слушатель умеет применять...»
4. **Литература старше 10 лет** — не актуальна
5. **Нет самостоятельной работы** — нарушает баланс часов
6. **Формулировки без таксономии Блума** — невозможно оценить результаты
7. **Фонды ОС не привязаны к модулям** — нет проверяемых критериев
8. **Отсутствие кода по ОКСП** — невозможно включить в реестр программ
9. **Двойной учёт часов** — один и тот же час в разных модулях
10. **Удостоверение при объёме < 16 часов** — несоответствие требованиям

## Pipeline разработки ДПО-программы

```
Концепция → Рабочая программа → УМК → Нормоконтроль → Экспертиза → Утверждение → Публикация

Phase 1: meta.json + concept/ (rationale, target-audience, charter)
Phase 2: program/ (working-program, curriculum, content, timeline)
Phase 3: umk/ (umk-full, teacher-guide, assessment)
Phase 4: docs/ (contract, application, certificate)
Phase 5: нормоконтроль (dpo_validator.py)
Phase 6: экспертиза (комиссия)
Phase 7: публикация в перечне
```

### Работа с контекстом

- При разработке каждого раздела — загружать только нужные reference файлы
- Не загружать все reference одновременно
- Использовать meta.json для согласованности данных между разделами
- После каждого этапа — запускать валидатор
- Перед экспертной оценкой — проверять чек-лист нормоконтроля

### Файлы тем (topic-*.md)

Каждая тема программы оформляется как отдельный markdown-файл в `program/`:

```
dpo-project/
├── meta.json                      # Метаданные программы
├── program/
│   ├── topic-1.md                 # Тема 1 (автоматически)
│   ├── topic-2.md                 # Тема 2 (автоматически)
│   └── ...
├── output/
│   └── program.docx               # Итоговый документ
```

**Структура файла темы (topic-N.md):**

```markdown
# Тема N. [Название темы]

**Общий объём:** X академических часов (Y аудиторных + Z самостоятельная работа)
**Формируемые компетенции:** ОПК-X, ПСК-Y

## N.1. Цели и задачи темы

### Цель
[Одна формулировка]

### Задачи
1. ...
2. ...

## N.2. Планируемые результаты обучения

### По таксономии Блума

| Результат | Глагол действия | Уровень | Индикатор |
|---|---|---|---|
| Знает | Перечисляет | 1-2 | ... |
| Умеет | Анализирует | 3-4 | ... |
| Владеет | Применяет | 5-6 | ... |

### Формируемые компетенции

| Код | Наименование | Тип |
|---|---|---|
| ОПК-X | ... | профессиональная |

## N.3. Содержательная часть темы

### Тема N.1. [Подтема] (X акад. часов)

**Форма:** Лекция / Практика / Семинар

**Содержание:**
- ...

**Результаты:**
- Слушатель знает ...
- Слушатель умеет ...

## N.4. Самостоятельная работа (X акад. часов)

**Задание:** ...

**Критерии оценивания:**

| Критерий | Отлично (5) | Хорошо (4) | Удовл. (3) |
|---|---|---|---|
| ... | ... | ... | ... |

## N.5. Контрольные вопросы

1. ...
2. ...

## N.6. Рекомендуемая литература

### Основная
1. ...

### Дополнительная
1. ...

### Интернет-ресурсы
- ...
```

**Правила оформления:**
- H1 (`#`) — название темы (с page break в docx)
- H2 (`##`) — разделы темы (N.1, N.2, ...)
- H3 (`###`) — подразделы (N.3.1, N.3.2, ...)
- Таблицы в Markdown-формате
- Списки через `—` (не `*` — конфликтует с курсивом)
- **жирный** и *курсив* поддерживаются

### Генерация docx

```bash
# Генерация из всех topic-*.md в папке program/
python scripts/dpo_md_to_docx.py \
  --meta dpo-project/meta.json \
  --all \
  --output dpo-project/output/program.docx \
  --base-dir dpo-project

# Генерация из конкретных тем
python scripts/dpo_md_to_docx.py \
  --meta dpo-project/meta.json \
  --topics dpo-project/program/topic-1.md dpo-project/program/topic-3.md \
  --output dpo-project/output/program.docx \
  --base-dir dpo-project
```

**Настройки docx:**
- Times New Roman, 14pt
- Интервал 1.5
- Красная строка 1.25 см
- Таблицы с границами и заголовками
- Оглавление (TOC-поле, обновляется по F9)
- Нумерация страниц (правый верхний угол, со 2-й страницы)
- Титульный лист с метаданными из meta.json
