# РДП — Разработка рабочих программ дисциплин

Агентская система для разработки рабочих программ дисциплин (РДП) в соответствии
с ФГОС ВО/СПО, учебными планами образовательных программ и требованиями
аккредитационной комиссии. Покрывает полный цикл от концепции до утверждения.

---

## Нормативная база

### Ключевые документы

1. **ФГОС ВО** (приказы Минобрнауки РФ: 346, 1034, 665 — для разных направлений)
2. **ФГОС СПО** (приказы Минобрнауки РФ: 350, 588)
3. **СанПиН 2.4.3648-20** — санитарные правила для образовательных организаций
4. **ГОСТ Р 7.0.100-2018** — библиографические записи
5. **ГОСТ Р ИСО 9001-2015** — система менеджмента качества
6. **Письма Минобрнауки** о структуре РПД
7. **Локальные акты вуза/колледжа** (университетские стандарты)

### Типы программ дисциплин

| Тип | Документ | Применение |
|---|---|---|
| Рабочая программа дисциплины (РПД / РДП) | РПД | Базовый документ |
| Рабочая программа учебной дисциплины | РПД (УД) | СПО |
| Рабочая программа учебной практики | РПД (П) | Практики |
| Рабочая программа факультатива | РПД (Ф) | Факультативы |

### Ключевые элементы РДП

```
1. Паспорт дисциплины (объём, часы, код, ЦА)
2. Планируемые результаты (ОПК, ПК, УПК по ФГОС)
3. Содержание дисциплины (тематическое планирование)
4. Фонды оценочных средств
5. Учебно-методическое обеспечение
6. Кадровое обеспечение
7. Информационно-техническое обеспечение
8. Календарный план
9. Фонды компетенций (для аккредитации)
```

## Фаза 1: Инициализация проекта

### 1.1 Сбор требований

Собрать у пользователя:

1. **Тип программы**: РПД УД / РПД практики / РПД факультатив
2. **Название дисциплины**: (черновое)
3. **Уровень образования**: СПО / ВО (бакалавриат / магистратура)
4. **Направление/код специальности**: ФГОС-код направления
5. **Объём в академических часах**: аудиторные + самостоятельные
6. **Форма контроля**: экзамен / зачёт / ТЗ / портфолио
7. **Семестр/курсы реализации**: в учебном плане
8. **Преподаватель**: ФИО, степень, должность
9. **Требования ФГОС**: обязательные/факультативные элементы
10. **Особенности**: лабораторные, проекты, практики

## Фаза 2: Концепция

### 2.1 Обоснование актуальности (`concept/rationale.md`)

**Структура:**

1. **Контекст** — роль дисциплины в образовательной программе
2. **Проблема** — соответствие компетенциям ФГОС
3. **Связь с учебным планом** — место дисциплины в программе
4. **Предшествующие и последующие дисциплины** — логика обучения
5. **Соответствие ФГОС** — конкретные пункты

### 2.2 Связь с учебным планом (`concept/connection.md`)

**Описание связи:**

- Как дисциплина формирует компетенции
- Предварительные знания (предшествующие дисциплины)
- Последующие дисциплины (на основе данной)
- Практическая направленность
- Соотношение теоретической и практической частей

### 2.3 Положение о дисциплине (`concept/charter.md`)

**Разделы положения:**

1. Общие положения (назначение, нормативная база)
2. Цели и задачи дисциплины
3. Планируемые результаты (по ФГОС компетенциям)
4. Объём программы (академ. часы)
5. Формы обучения и контроля
6. Требования к результатам освоения
7. Требования к фонду оценочных средств
8. Финансовые условия (если применимо)

## Фаза 3: Рабочая программа

### 3.1 Структура рабочей программы

Согласно требованиям ФГОС и локальным актам вуза:

```
1. Паспорт дисциплины
   1.1. Наименование
   1.2. Код направления
   1.3. Целевая аудитория
   1.4. Объём (акад. часы)
   1.5. Форма обучения и контроля
   1.6. Семестр реализации

2. Планируемые результаты
   2.1. Формируемые компетенции (ОПК, ПК, УПК)
   2.2. Результаты (знает, умеет, владеет)
   2.3. Критерии оценки

3. Содержание дисциплины
   3.1. Раздел/модуль 1 — наименование, объём, содержание
   3.2. Раздел/модуль 2 — ...
   ...

4. Фонды оценочных средств (ФОС)

5. Учебно-методическое обеспечение
   5.1. Основная литература
   5.2. Дополнительная литература
   5.3. Интернет-ресурсы

6. Кадровое обеспечение

7. Информационно-техническое обеспечение

8. Финансово-экономические условия

9. Календарный план
```

### 3.2 Паспорт дисциплины (`program/working-program.md`)

| Целевая аудитория | Категория обучающихся |
| Объём (акад. часы) | XX |
| Аудиторные часы | XX |
| Самостоятельная работа | XX |

| Семестр | X |
| Преподаватель | ФИО, степень, должность |

### 3.3 Планируемые результаты (`program/content.md`)

**Формулировка компетенций по ФГОС:**

| Компетенция | Индикатор достижения | Уровень |
|---|---|---|
| ОПК-X | Применяет методы ... | уровень X (применение) |
| ПК-X | Анализирует данные ... | уровень X |
| УПК-X | Способен решать ... | уровень X |

**Результаты по формуле «знает-умеет-владеет»:**

- **Знает**: основные принципы, нормативную базу, классификации
- **Умеет**: применять методы, анализировать данные, принимать решения
- **Владеет**: навыками, технологиями, инструментами

**Для каждой дисциплины модули/разделы:**

## Фаза 4: Фонд компетенций (ключевой модуль)

### 4.1 Фонд компетенций (`competency/competencies.md`)

**Для каждой компетенции:**

- Код компетенции (ОПК, ПК, УПК)
- Наименование
- Индексы модулей/тем
- Индикаторы достижения
- Баллы/уровни
- Формы контроля
- Ответственный преподаватель

**Пример:**

| Код компетенции | Наименование | Индекс модуля | Индикатор | Балл |
|---|---|---|---|---|
| ОПК-1 | Способен ... | Модуль 1, 2 | Решает ... | X.XX |
| ПК-1 | Применяет ... | Модуль 2 | Выполняет ... | X.XX |

### 4.2 Матрица компетенций (`competency/matrix.md`)

**Связь дисциплина — компетенции — темы:**

| Тема | ОПК-1 | ОПК-2 | ПК-1 | ПК-2 |
|---|---|---|---|---|
| Тема 1.1 | X | + | + | — |
| Тема 1.2 | + | X | + | + |
| Тема 2.1 | — | + | X | + |

**Расшифровка:** X — формирование, + — закрепление, — — повторение

### 4.3 Фонд проверяемых компетенций (`competency/verification.md`)

**Для каждой компетенции:**

- Форма контроля (тест, проект, защита, экзамен)
- Критерии оценки
- Масштаб (1-10 баллов)
- Вес компетенции в итоговой оценке

## Фаза 5: Учебно-методический комплект

### 5.1 Структура УМК (`umk/umk-full.md`)

```
УМК дисциплины:
├── Для студента:
│   ├── Учебно-методическое описание
│   ├── Рабочая программа дисциплины
│   ├── Методические указания по выполнению работ
│   ├── Сборник заданий / кейсов
│   └── Рекомендации по самостоятельной работе
├── Для преподавателя:
│   ├── Рабочая программа
│   ├── Календарно-тематический план
│   ├── Методические указания
│   ├── Фонды оценочных средств
│   └── Примерный перечень вопросов к зачёту/экзамену
└── Оценочные средства:
    ├── Фонды ОС (тесты, кейсы, задания)
    ├── Критерии оценивания
    └── Протоколы аттестации
```

### 5.2 Фонды оценочных средств (`umk/assessment/`)

**Для каждой темы/модуля:**

- Тестовые задания (минимум 20 вариантов)
- Кейс-задачи (практические ситуации)
- Практические задания
- Промежуточная и итоговая аттестация

**Критерии оценивания:**

| Оценка | Критерий |
|---|---|
| Отлично (5) | 85-100% правильных ответов / высокий балл по компетенциям |
| Хорошо (4) | 70-84% |
| Удовлетворительно (3) | 50-69% |
| Неудовлетворительно (2) | менее 50% |

## Фаза 6: Календарный план и организация

### 6.1 Календарный план (`program/timeline.md`)

**Разбить программу на этапы:**

| Неделя | Тема | Форма | Часы | Результат |
|---|---|---|---|---|
| 1 | Модуль 1.1 | Лекция + семинар | 8 | Знание основ |
| 2 | Модуль 1.2 | Практикум | 8 | Навык применения |
| ... | | | | |
| N | Итоговый контроль | Экзамен | 4 | Оценка компетенций |

### 6.2 Организация учебного процесса

**Для очной формы:**
- Аудиторный фонд
- Расписание занятий
- Правила посещения (минимум 80% посещаемости)

**Для заочной формы:**
- Платформа ЭО
- Структура курса на платформе
- Формы взаимодействия с преподавателем
- Сроки сдачи заданий

## Фаза 7: Нормоконтроль

### 7.1 Чек-лист нормоконтроля

| Параметр | Требование | Проверка |
|---|---|---|
| Код по ФГОС | Соответствует направлению | [ ] |
| Объём | Соответствует ФГОС (мин.) | [ ] |
| Баланс часов | Ауд. + самост. = общий объём | [ ] |
| Фонды ОС | Соответствуют каждому модулю | [ ] |
| Фонд компетенций | Все компетенции охвачены | [ ] |
| Матрица компетенций | Корректная связь темы → компетенция | [ ] |
| Литература | Не старше 5 лет (не менее 30% за 3 года) | [ ] |
| Квалификация преподавателей | Наличие учёной степени / опыта | [ ] |
| Оформление | ГОСТ Р 7.0.100-2018 | [ ] |
| Номер лицензии | Указан в программе | [ ] |

### 7.2 Экспертная оценка

**Комиссия включает:**

1. Эксперт по содержанию (предметная область)
2. Методист (педагогический дизайн)
3. Представитель методической комиссии
4. Аккредитационный эксперт (при необходимости)

**Критерии экспертизы:**

| Критерий | Вес | Описание |
|---|---|---|
| Актуальность | 0.25 | Соответствие запросам отрасли |
| Полнота | 0.20 | Охват всех компетенций |
| Методическая грамотность | 0.20 | Соответствие педагогическим стандартам |
| Практическая значимость | 0.20 | Применимость в практике |
| Оформление | 0.15 | Соответствие ГОСТ |
| Фонд компетенций | 0.20 | Корректность матрицы |

**Минимальный порог:** 4.0 из 5.0

```
1. Разбить на темы с указанием часов
2. Для каждой темы определить форму
3. Сформулировать результаты по Bloom's taxonomy
4. Указать виды самостоятельной работы
5. Проверить баланс: ауд. часы + самост. = общий объём
6. Привязать темы к компетенциям

Выход: Markdown-таблица + текстовое описание
```

### Промпт для генерации фонда компетенций

```
Ты — эксперт по ФГОС. Разработай фонд компетенций для дисциплины.

Дано:
- Направление: [код/название]
- Коды формируемых компетенций: [ОПК-X, ПК-Y]
- Темы дисциплины: [перечень]
- Формы контроля: [зачёт/экзамен/тз]

Требования:
1. Указать индекс компетенций по модулям/темам
2. Определить индикаторы достижения
3. Привязать формы контроля к компетенциям
4. Проверить полноту охвата всех компетенций

Выход: JSON с полями компетенций + Markdown-матрица
```

### Промпт для нормоконтроля

```
Ты — эксперт по нормоконтролю РДП.

Проверь рабочую программу на соответствие:
1. ФГОС (минимальный объём, требования к компетенциям)
2. Баланс часов (аудиторные + самостоятельные = заявленные)
3. Таксономия Блума (формулировки результатов)
4. Фонд компетенций (все компетенции охвачены)
5. Актуальность литературы (не старше 5 лет)

Результат: JSON с полями:
- passed: boolean
- issues: [{line, type, severity, description}]
- warnings: [{line, type, description}]
- recommendations: [текст]
```

## Антипаттерны РДП-программ

1. **Объём менее 72 часов** — дисциплина не соответствует требованиям ФГОС
2. **Компетенции без привязки к темам** — невозможно оценить формирование
3. **Литература старше 10 лет** — не актуальна
4. **Нет самостоятельной работы** — нарушает баланс часов
5. **Формулировки без таксономии Блума** — невозможно оценить результаты
6. **Фонды ОС не привязаны к модулям** — нет проверяемых критериев
7. **Отсутствие матрицы компетенций** — критично для аккредитации
8. **Двойной учёт часов** — один и тот же час в разных модулях
9. **Матрица компетенций с ошибками** — компетенция без индекса или дублирование
10. **Компетенции без весов** — невозможно рассчитать итоговый балл

## Pipeline разработки РДП-программы

```
Концепция → Рабочая программа → Фонд компетенций → УМК → Нормоконтроль → Экспертиза → Утверждение → Публикация

Phase 1: meta.json + concept/ (rationale, connection, charter)
Phase 2: program/ (working-program, curriculum, content, timeline)
Phase 3: competency/ (competencies, matrix, verification)
Phase 4: umk/ (umk-full, teacher-guide, assessment)
Phase 5: нормоконтроль (rdp_validator.py)
Phase 6: экспертиза (комиссия)
Phase 7: публикация в реестре
```

### Работа с контекстом

- При разработке каждого раздела — загружать только нужные reference файлы
- Не загружать все reference одновременно
- Использовать meta.json для согласованности данных между разделами
- После каждого этапа — запускать валидатор
- Перед экспертной оценкой — проверять чек-лист нормоконтроля

## Файлы тем (topic-*.md)

Каждая тема дисциплины оформляется как отдельный markdown-файл в `program/`:

```
rdp-project/
├── meta.json                      # Метаданные дисциплины
├── program/
│   ├── topic-1.md                 # Тема 1 (автоматически)
│   ├── topic-2.md                 # Тема 2 (автоматически)
│   └── ...
├── output/
│   └── rdp.docx                   # Итоговый документ
```

**Структура файла темы (topic-N.md):**

```markdown
# Тема N. [Название темы]

**Общий объём:** X академических часов (Y аудиторных + Z самостоятельная работа)
**Формируемые компетенции:** ОПК-X, ПК-Y

## N.1. Цели и задачи темы

### Цель
[Одна формулировка]

### Задачи
1. ...
2. ...

## N.2. Планируемые результаты обучения

### По таксономии Блума

| Результат | Глагол действия | Уровень | Индикатор |
|---|---|---|---|
| Знает | Перечисляет | 1-2 | ... |
| Умеет | Анализирует | 3-4 | ... |
| Владеет | Применяет | 5-6 | ... |

### Формируемые компетенции

| Код | Наименование | Тип |
|---|---|---|
| ОПК-X | ... | профессиональная |

## N.3. Содержательная часть темы

### Тема N.1. [Подтема] (X акад. часов)

**Форма:** Лекция / Практика / Семинар

**Содержание:**
- ...

**Результаты:**
- Обучающийся знает ...
- Обучающийся умеет ...

## N.4. Самостоятельная работа (X акад. часов)

**Задание:** ...

**Критерии оценивания:**

| Критерий | Отлично (5) | Хорошо (4) | Удовл. (3) |
|---|---|---|---|
| ... | ... | ... | ... |

## N.5. Контрольные вопросы

1. ...
2. ...

## N.6. Рекомендуемая литература

### Основная
1. ...

### Дополнительная
1. ...

### Интернет-ресурсы
- ...
```

**Правила оформления:**
- H1 (`#`) — название темы (с page break в docx)
- H2 (`##`) — разделы темы (N.1, N.2, ...)
- H3 (`###`) — подразделы (N.3.1, N.3.2, ...)
- Таблицы в Markdown-формате
- Списки через `—` (не `*`)
- **жирный** и *курсив* поддерживаются

### Генерация docx

```bash
# Генерация из всех topic-*.md в папке program/
python scripts/rdp_md_to_docx.py \
  --meta rdp-project/meta.json \
  --all \
  --output rdp-project/output/rdp.docx \
  --base-dir rdp-project

# Генерация из конкретных тем
python scripts/rdp_md_to_docx.py \
  --meta rdp-project/meta.json \
  --topics rdp-project/program/topic-1.md rdp-project/program/topic-3.md \
  --output rdp-project/output/rdp.docx \
  --base-dir rdp-project
```

**Настройки docx:**
- Times New Roman, 14pt
- Интервал 1.5
- Красная строка 1.25 см
- Таблицы с границами и заголовками
- Оглавление (TOC-поле, обновляется по F9)
- Нумерация страниц (правый верхний угол, со 2-й страницы)
- Титульный лист с метаданными из meta.json
