# Headroom Compressor

Слой сжатия контекста для AI-агентов. Позволяет подавать большие объемы логов, файлов и истории диалогов без превышения лимитов и с экономией токенов.

## Режимы работы

1. **Proxy Mode (Прокси):**
   Запуск локального прокси-сервера:
   ```bash
   headroom proxy --port 8787
   ```
   После этого перенаправьте запросы агента/клиента на `http://localhost:8787/v1`.

2. **Agent Wrap (Обертка):**
   Обернуть сессию существующего CLI-агента (например, Claude Code):
   ```bash
   headroom wrap claude
   ```

3. **Inline Library (Библиотека):**
   *   **Python:**
       ```python
       from headroom import compress
       compressed_messages = compress(messages)
       ```
   *   **Node.js / TypeScript:**
       ```javascript
       import { compress } from 'headroom-ai';
       const compressed = await compress(messages);
       ```

4. **MCP Server:**
   Подключение MCP-сервера Headroom для внешних клиентов через инструменты `headroom_compress`, `headroom_retrieve` и `headroom_stats`.

## Архитектура сжатия

*   **ContentRouter:** Определяет тип входящего контента и перенаправляет его на соответствующий алгоритм сжатия.
*   **SmartCrusher:** Сжатие JSON-структур.
*   **CodeCompressor:** AST-ориентированное сжатие исходного кода.
*   **Kompress-base:** Сжатие прозы и естественного языка на базе LLM.
*   **CacheAligner:** Выравнивание префиксов для обеспечения корректной работы KV-кэша LLM-провайдеров.

---

## Reversible Compression (CCR)

При сильном сжатии детали могут быть утеряны. Для этого Headroom кэширует оригинальные тексты локально.
*   Если модели во время рассуждения требуется точный исходный фрагмент, она вызывает инструмент `headroom_retrieve(key)` для получения исходного контента.
*   Не удаляйте кэш и не переопределяйте `DATABASE_URL` вручную, так как это нарушит связь с кэшем CCR.

## Обучение и коррекция (`headroom learn`)

Запустите анализатор сессий для поиска ошибок контекста и автоматического внесения исправлений в инструкции (`CLAUDE.md` / `AGENTS.md`):
```bash
headroom learn
```
Команда проанализирует неудачные вызовы и предложит паттерны для сокращения контекста в будущих сессиях.
