# PARA Memory (Файловая память)

Поскольку контекст сессии сбрасывается при перезапуске, долгосрочная память агента организуется в текстовых файлах в директории `$AGENT_HOME` по методологии PARA (Projects, Areas, Resources, Archives).

## Три уровня памяти

1.  **Граф знаний (Layer 1 — PARA):**
    Располагается в папке `$AGENT_HOME/life/`. Каждая сущность (проект, сфера ответственности, ресурс) получает свою папку с двумя файлами:
    *   `summary.md` — краткое текстовое описание (загружается первым).
    *   `items.yaml` — список атомарных фактов (с датой, статусом и связями).

    Категории PARA:
    *   `projects/` — активные задачи со сроками/целями.
    *   `areas/` — постоянные зоны ответственности (люди, компании, хобби).
    *   `resources/` — справочные материалы, темы для изучения.
    *   `archives/` — неактивные сущности, перенесенные из верхних категорий.

2.  **Дневники (Layer 2 — Daily Notes):**
    Файлы временной шкалы по пути `$AGENT_HOME/memory/YYYY-MM-DD.md`. Сюда записываются все ключевые события дня, которые затем (при завершении сессии или во время heartbeats) структурируются и переносятся в Граф знаний.

3.  **Неявные знания (Layer 3 — Tacit Knowledge):**
    Файл `$AGENT_HOME/MEMORY.md`. Содержит паттерны работы пользователя, его личные предпочтения, правила общения и выводы из прошлых ошибок.

---

## Правила работы с фактами
*   Не удаляйте факты. Если факт устарел, измените его статус на `superseded` и укажите ссылку на новый факт в поле `superseded_by`.
*   Всегда отдавайте приоритет записи на диск перед удержанием информации в голове.

---

## Поиск и индексация через `qmd`

Для быстрого поиска по памяти используйте утилиту `qmd`:
*   **Семантический поиск с реранкингом:**
    ```bash
    qmd query "что произошло на прошлой неделе"
    ```
*   **Поиск по ключевым словам (BM25):**
    ```bash
    qmd search "ключевая фраза"
    ```
*   **Векторный поиск по концептам:**
    ```bash
    qmd vsearch "описание концепта"
    ```
*   **Индексация памяти:** `qmd index $AGENT_HOME`
