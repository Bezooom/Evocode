# Практический план внедрения Skill Sync Engine

## Overview

Этот план описывает пошаговый процесс внедрения Skill Sync Engine — компонента «Эвокод», обеспечивающего ежедневную автоактуализацию навыков из GitHub.

---

## Этапы внедрения

### Этап 1: Создание каталогов (1 час)

**Задачи:**
- [ ] Создать `skills/system/` для системных навыков
- [ ] Создать `skills/user/` для пользовательских оверрайдов
- [ ] Создать `skills/.backup/` для бэкапов
- [ ] Создать `skills/.archive/` для удалённых навыков

**Результат:** Готовая файловая структура для навыков.

---

### Этап 2: Настройка источников синхронизации (2-3 часа)

**Задачи:**
- [ ] Создать `sync-sources.json` с источниками
- [ ] Добавить GitHub-репозитории (evocode/skills, kilocode/skills)
- [ ] Настроить branch и path для каждого источника
- [ ] Проверить доступность источников

**Результат:** Настроенные источники синхронизации.

---

### Этап 3: Реализация парсинга GitHub (1-2 дня)

**Задачи:**
- [ ] Реализовать функцию получения дерева файлов из GitHub API
- [ ] Реализовать функцию получения последнего коммита
- [ ] Реализовать функцию сравнения content-hash
- [ ] Настроить кэширование для уменьшения нагрузки на GitHub API

**Результат:** Функция парсинга GitHub работает корректно.

---

### Этап 4: Реализация diff и применения (1-2 дня)

**Задачи:**
- [ ] Реализовать логику diff (new, changed, removed, unchanged)
- [ ] Реализовать логику применения изменений
- [ ] Реализовать soft-delete для удалённых навыков
- [ ] Реализовать бэкап предыдущей версии

**Результат:** Навыки синхронизируются корректно.

---

### Этап 5: Реализация верификации (1 день)

**Задачи:**
- [ ] Реализовать проверку подписи/хеша источника
- [ ] Реализовать валидацию структуры SKILL.md
- [ ] Реализовать проверку зависимостей
- [ ] Настроить обработку ошибок

**Результат:** Навыки верифицируются перед применением.

---

### Этап 6: Реализация логирования (1 день)

**Задачи:**
- [ ] Реализовать логику логирования операций
- [ ] Создать формат лога (timestamp, операция, результат)
- [ ] Реализовать ротацию логов
- [ ] Настроить запись в sync.log

**Результат:** Все операции синхронизации логируются.

---

### Этап 7: Реализация UI (2-3 дня)

**Задачи:**
- [ ] Создать панель «Навыки»
- [ ] Добавить список навыков с статусами

- [ ] Добавить вкладку «Лог синхронизации»

## Что нужно для начала

### 1. GitHub аккаунт

- Создать репозитории:
  - `github.com/evocode/skills`
  - `github.com/kilocode/skills`

### 2. GitHub API Token

- Создать token в Settings → Developer settings → Personal access tokens
- Выдать права: `repo`, `read:org`

### 3. Настроить sync-sources.json

```json
{
  "sources": [
    {
      "name": "evocode-skills",
      "url": "https://github.com/evocode/skills",
      "path": "skills/",
      "branch": "main",
      "type": "github"
    }
  ]
}
```

## Что получится в итоге

### Автоматизация

- **Ежедневная синхронизация** навыков из GitHub
- **Автоматическое обновление** новых и изменённых навыков
- **Soft-delete** для удалённых навыков
- **Бэкап** предыдущей версии перед обновлением

### Надёжность

- **Верификация** перед применением
- **Обработка breaking changes**
- **Офлайн-поведение** при недоступности сети
- **Логирование** всех операций

### Удобство

- **UI для управления** навыками
- **Откат** к предыдущей версии
- **Лог синхронизации** с историей изменений
