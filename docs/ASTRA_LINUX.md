# 🐧 Руководство по установке и запуску Эвокод на ОС Astra Linux

Данный документ описывает процесс развертывания, установки и верификации среды разработки **Эвокод** в операционных системах **Astra Linux Common Edition 2.12** и **Astra Linux Special Edition 1.7** (включая режим усиленного контроля безопасности Parsec).

---

## 1. Системные требования и зависимости

Для успешного запуска графической оболочки IDE (VSCodium) и локального ИИ-рантайма на Astra Linux требуются следующие системные библиотеки:

*   **Графический стек и ядро:** `libnss3`, `libatk1.0-0`, `libatk-bridge2.0-0`, `libcups2`, `libdrm2`, `libgtk-3-0`, `libgbm1`, `libasound2`
*   **Изоляция процессов (песочница):** утилита `bubblewrap` (пакет `bubblewrap` версии `0.4.0+`)
*   **Среда выполнения:** Node.js (рекомендуется v18.x или v20.x LTS)
*   **Инференс:** утилиты компиляции gcc/g++ (для локальной сборки llama.cpp при необходимости)

---

## 2. Установка из Debian-пакета (.deb)

Deb-пакет Эвокод содержит скомпилированную и брендированную сборку среды, исполняемый файл запуска `/usr/bin/evocode`, ярлык рабочего стола и необходимые ассеты.

### Шаг 2.1. Установка пакета

**Вариант A — deb (предпочтительно):**

```bash
# после npm run ide:package-portable && npm run ide:package-deb
sudo dpkg -i packages/ide/dist/evocode_*_amd64.deb
sudo apt-get install -f   # при необходимости
evocode
```

**Вариант B — portable tar.gz (полный продукт, не plain VSCodium):**

```bash
tar -xzf packages/ide/dist/releases/evocode-linux-x64-*.tar.gz -C ~/Evocode
~/Evocode/bin/evocode
```

Внутри: branded IDE + `evocode-agent` + `evocode-shell` + `core/` (см. [PACKAGING.md](PACKAGING.md)).

### Шаг 2.2. Разрешение зависимостей
Если при установке deb возникли ошибки из-за отсутствующих системных библиотек:
```bash
sudo apt-get install -f
```

## 3. Настройка безопасности и песочницы (Bubblewrap)

Astra Linux Special Edition содержит встроенные механизмы мандатного контроля доступа (LSM Parsec). Использование встроенной песочницы `bwrap` позволяет запускать дочерние процессы агента в изолированном пространстве имен без нарушения политик безопасности ОС.

### Шаг 3.1. Установка Bubblewrap
Установите пакет из официального репозитория Astra Linux:
```bash
sudo apt-get install bubblewrap
```

### Шаг 3.2. Активация песочницы в Эвокод
1.  Запустите **Эвокод**.
2.  Откройте единую панель настроек (комбинация клавиш `Ctrl+Shift+M` или перейдите в меню статус-бара `Модели`).
3.  Перейдите на вкладку **Агент**.
4.  В разделе **Безопасность** установите флаг **"Изолированная песочница (bubblewrap)"** в активное состояние (`true`).
5.  Сохраните настройки. Теперь все запускаемые ИИ-агентом консольные команды будут выполняться внутри легковесного контейнера с файловой системой «только для чтения» (доступ на запись разрешен только для папки текущего проекта и локальных конфигов `.config/evocode`).

## 4. Верификационные тесты (Smoke-тестирование)

Для быстрой проверки работоспособности всех компонентов (Core, DLP Guard, локальный инференс, плагины) на Astra Linux запустите встроенный автоматический тест:

```bash
npm run test:smoke
```
